## 分布式系统核心概念

### 时间与顺序 (Time and Order)

**物理时钟问题：**
- **Clock Skew（时钟偏移）**：不同处理器时钟值的差异
- **Clock Drift（时钟漂移）**：时钟相对于真实时间的变化率
- NTP同步无法完全消除偏移

**逻辑时钟：**
- **Lamport Timestamps**：
  - 每个事件关联一个逻辑时间戳
  - 偏序关系：→（happens-before）
  - 规则：
    1. 同一进程中，每个事件前加1
    2. 发送消息时，时间戳+1
    3. 接收时，取max(本地时间, 消息时间戳)+1
  
- **Vector Clocks**：
  - 每个进程维护一个向量
  - 向量[i] = 进程i已发生的事件数
  - 比Lamport Timestamp更能检测并发

### 互斥算法

**Ricart-Agrawala算法：**
- 全分布式互斥算法
- 需要2(N-1)条消息
- 基于Lamport逻辑时钟

**Maekawa投票算法：**
- 改进的投票算法
- 需要√N量级消息
- 使用 quorum 机制

### 共识问题

**CAP定理：**
- Consistency（一致性）
- Availability（可用性）
- Partition Tolerance（分区容错性）
- 只能同时满足两个！

**Raft算法实现细节：**

**Leader Election：**
1. 选举超时（Election Timeout）：150-300ms随机
2. 转换为Candidate，增加Term
3. 给自己投票，发送RequestVote
4. 获得多数票则成为Leader

**Log Replication：**
1. Leader接收客户端请求
2. 添加到本地日志
3. 发送AppendEntries给Followers
4. 多数确认后commit
5. 应用到状态机

**安全性保证：**
- Election Safety：同一Term最多一个Leader
- Leader Append-Only：Leader从不覆盖或删除日志
- Log Matching：如果两个日志有相同索引和Term，则日志相同
- Leader Completeness：当选Leader包含所有committed日志
- State Machine Safety：应用相同顺序的日志产生相同状态

## 并行计算核心概念

### OpenMP详解

**并行区域 (Parallel Region)：**
```c
#pragma omp parallel
{
    // 并行执行
}
```

**工作共享 (Work Sharing）：**
```c
#pragma omp for
for (int i = 0; i < N; i++) {
    // 并行处理
}

#pragma omp sections
{
    #pragma omp section
    // 任务1
    
    #pragma omp section
    // 任务2
}
```

**数据作用域 (Data Scope)：**
- `shared`：所有线程共享
- `private`：每个线程独立副本
- `firstprivate`：私有但初始化为原值
- `reduction`：归约操作

**同步：**
- `barrier`：所有线程等待
- `critical`：互斥访问
- `atomic`：原子操作

### CUDA详解

**内存层次：**
- Global Memory：最大但最慢
- Shared Memory：每个block共享，快速
- Register：最快，每个线程私有
- Constant/Texture：只读，缓存

**优化技巧：**
- 合并内存访问 (Coalesced Memory Access)
- 避免bank conflict
- 使用shared memory
- 合理设置block大小

### MPI详解

**点对点通信：**
- `MPI_Send`：同步发送
- `MPI_Isend`：异步发送
- `MPI_Recv`：接收
- `MPI_Sendrecv`：双向通信

**集合通信：**
- `MPI_Broadcast`：广播
- `MPI_Scatter`：分发
- `MPI_Gather`：收集
- `MPI_Reduce`：归约
- `MPI_Allreduce`：全归约
