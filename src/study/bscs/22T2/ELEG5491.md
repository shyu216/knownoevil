## 📚 课程笔记 (Course Notes)

### 第一部分：神经网络基础

---

### Q1: 什么是计算图？它在深度学习中的作用是什么？

**Answer:**

**计算图（Computational Graph）** 是用图形方式表示函数组合的概念，每个节点代表一个变量（标量、向量、矩阵等），每条边代表一个操作。

**核心作用**：
- **前向计算（Forward Computation）**：从底到顶计算分类概率
- **反向传播（Back-propagation）**：从顶到底计算梯度，用于参数更新

**示例**：
```
u = bc, v = a + u, J = 3v

前向计算：给定输入，计算输出
反向计算：沿计算图反向计算梯度，避免冗余计算
```

**链式法则应用**：
```
∂J/∂v = ∂J/∂a · ∂a/∂v
∂J/∂u = ∂J/∂v · ∂v/∂u
∂J/∂b = ∂J/∂u · ∂u/∂b
∂J/∂c = ∂J/∂u · ∂u/∂c
```

---

### Q2: 什么是全连接层（Fully-Connected Layer）？

**Answer:**

**全连接层** 是神经网络的基本结构块，计算公式为：
```
y = Wx + b
```
其中：
- W ∈ R^(C×n) 是权重矩阵
- b ∈ R^C 是偏置向量
- x ∈ R^n 是输入特征向量
- y ∈ R^C 是输出向量

**矩阵表示**：
```
┌                    ┐   ┌     ┐   ┌      ┐
│ w11  w12  ···  w1n │   │  b1 │   │  x1  │
│ w21  w22  ···  w2n │   │  b2 │   │  x2  │
│  ··   ··   ··   ·· │ × │  ·  │ + │  ·  │ = y
│ wC1  wC2  ···  wCn │   │  bC │   │  xn  │
└                    ┘   └     ┘   └      ┘
```

**梯度计算**：
- ∂J/∂W = (∂J/∂y) · x^T （外积）
- ∂J/∂b = ∂J/∂y
- ∂J/∂x = W^T · (∂J/∂y)

---

### Q3: 什么是激活函数？有哪些常见类型？

**Answer:**

**激活函数** 为网络引入非线性，是深度神经网络能够逼近任意复杂函数的关键。

**Sigmoid函数**：
```
σ(z) = 1 / (1 + e^(-z))
```
- 输出范围：[0, 1]
- 用于二分类输出层
- 缺点：梯度消失问题（导数最大0.25）

**Tanh函数**：
```
tanh(z) = (e^z - e^(-z)) / (e^z + e^(-z))
```
- 输出范围：[-1, 1]
- 零中心化
- 缺点：仍有梯度消失问题

**ReLU函数**（最常用）：
```
y = max(0, x)
```
- 计算高效，收敛快
- 缺点：神经元死亡问题（输入小于0时梯度为0）

**Leaky ReLU**：
```
y = αx if x < 0
y = x    if x ≥ 0
```
- 解决ReLU的神经元死亡问题
- α通常设为0.01

**PReLU**：
- Leaky ReLU的α变为可学习参数
- 前向和后向计算时根据输入动态调整

---

### Q4: Softmax层和交叉熵损失如何工作？

**Answer:**

**Softmax函数**：
```
pk = exp(yk) / Σexp(yi)  (i from 1 to C)
```
- 将任意实数向量转换为概率分布
- 所有输出之和为1

**交叉熵损失**：
```
J = -Σ(ŷi · log(pi))
```
- 当真实标签为1时，最大化对应概率
- 当真实标签为0时，最小化对应概率

**梯度计算**：
```
if i = j:  ∂J/∂pi = -1/pi
if i ≠ j:  ∂J/∂pi = 0
```

**Softmax + Cross-Entropy组合的梯度**：
```
当 ŷi = 1时:
- 如果i=j:  ∂J/∂yj = -(1 - pi) = pi - 1
- 如果i≠j: ∂J/∂yj = pj
```

**重要性质**：这个组合在数学上等价于将梯度直接传回yj，简化计算！

---

### Q5: 为什么需要多层神经网络？

**Answer:**

**理论背景**：
- 三层神经网络理论上可以逼近任意非线性函数
- 逻辑回归可以看作"浅层"三层神经网络

**为什么需要"深度"网络？**：

| 方案 | 问题 |
|------|------|
| 浅层网络 | 如果函数非常复杂，需要**指数级**增长的神经元数量 |
| 深度网络 | 通过多层结构，每层少量神经元就能逼近复杂函数 |

**逐层特征学习**：
- 低层特征：捕捉通用的模式（边缘、纹理）
- 高层特征：更接近最终任务的语义信息
- 例如：图像 → 边缘 → 纹理 → 物体部件 → 完整物体

**MLP结构**：
```
输入层 → 隐藏层1 → 隐藏层2 → ... → 隐藏层N → 输出层
  x      W1x+b1     W2x+b2              WNx+bN      y
```

---

### Q6: 什么是批归一化（Batch Normalization）？

**Answer:**

**核心思想**：对每个mini-batch的输入特征进行标准化

**前向计算**：
```
μB = mean(x)  over batch B
σ²B = variance(x) over batch B
x_norm = (x - μB) / sqrt(σ²B + ε)
y = γ ⊙ x_norm + β
```
其中γ and β是可学习的缩放和偏移参数

**作用**：
1. **加速训练**：允许使用更高的学习率
2. **减轻梯度消失**：使梯度传播更稳定
3. **对初始化不那么敏感**：不需要小心翼翼地选择初始权重
4. **支持更多激活函数**：例如Sigmoid也能有效使用

**训练vs推理**：
- 训练：使用当前mini-batch的均值和方差
- 推理：使用训练过程中累积的运行均值和方差

---

### Q7: 什么是Dropout？为什么能防止过拟合？

**Answer:**

**Dropout机制**：
- 训练时：随机将比例为p的神经元置零
- 推理时：所有神经元参与，但输出乘以(1-p)

**直观理解**：
- 强迫网络不依赖少数神经元
- 每个神经元必须学会独立做出好的预测
- 相当于训练了多个不同结构的网络并做集成

**为什么有效**：
- 减少神经元之间的共适应（co-adaptation）
- 增加网络的鲁棒性
- 类似L2正则化的效果

**使用建议**：
- 通常放在全连接层之后
- 不要放在最后一层（全连接层）之后
- 在大数据集上效果不明显

## 第二部分：卷积神经网络（CNN）

---

### Q8: 全连接网络处理图像有什么问题？

**Answer:**

**问题1：参数过多**
- 假设输入图像1000×1000×3 = 3M像素
- 第一个隐藏层1000个神经元
- 参数量：3M × 1000 = 30亿（不可行）

**问题2：丢失空间结构**
- 将图像展平后，像素的位置信息丢失
- 相邻像素的相关性无法被利用

**问题3：不平移不变**
- 图像中的物体移动后，全连接网络难以正确识别

---

### Q9: 卷积神经网络的核心思想是什么？

**Answer:**

**三大核心原则**：

**1. 局部连接（Local Connectivity）**
- 每个神经元只连接输入的一个局部区域
- 这个局部区域称为"感受野"（Receptive Field）
- 类似于生物学中的视觉皮层细胞

**2. 权重共享（Weight Sharing）**
- 同一层的神经元共享相同的权重
- 大大减少参数数量
- 实现平移不变性

**3. 多层堆叠**
- 低层捕捉局部特征（边缘、纹理）
- 高层捕捉全局特征（物体部件、完整物体）

---

### Q10: 2D卷积是如何工作的？

**Answer:**

**卷积操作**：
```
y(i,j) = Σ Σ w(u,v) · x(i+u, j+v) + b
          u=-k/2 v=-k/2
```

**输入/输出**：
- 输入特征图：x ∈ R^(P×Q)
- 卷积核：w ∈ R^(k×k)
- 输出特征图：y ∈ R^(P'×Q')

**关键参数**：

| 参数 | 说明 |
|------|------|
| **Kernel Size** | 卷积核大小（通常3×3, 5×5, 7×7） |
| **Stride** | 滑动步长（步长2会使输出尺寸减半） |
| **Padding** | 边缘填充（保持空间尺寸） |

**输出尺寸公式**：
```
Lout = floor((Lin + 2×padding - dilation×(kernel_size-1) - 1) / stride + 1)
```

---

### Q11: 什么是多通道卷积？

**Answer:**

**多输入通道**：
- 输入特征图：x ∈ R^(P×Q×C)，C是通道数
- 卷积核扩展为：w ∈ R^(k×k×C)
- 每个通道有独立的卷积核权重

**多输出通道**：
- 多个卷积核生成多个输出通道
- D个卷积核 → D维输出特征图
- 特征图沿通道维度拼接

**分组卷积**：
- 将输入通道分成多个组
- 每组使用独立的卷积核
- 减少计算量和参数量
- MobileNet、ResNeXt等架构的基础

---

### Q12: 什么是空洞卷积（Dilated Convolution）？

**Answer:**

**核心思想**：在卷积核中加入"空洞"，在不增加参数的情况下扩大感受野

**空洞率（Dilation Rate）**：
- 空洞率l表示相邻权重之间间隔l个像素
- k×k核，有效覆盖范围 = k + (k-1)×(l-1)

**示例**：
- 3×3核，空洞率1：覆盖3×3区域
- 3×3核，空洞率2：覆盖5×5区域（但仍只有9个参数）

**应用场景**：
- 需要大感受野的任务（语义分割、语音合成）
- 在不损失分辨率的情况下扩大感受野

---

### Q13: 什么是反卷积（Transposed Convolution）？

**Answer:**

**用途**：上采样，扩大空间尺寸

**常见场景**：
- 语义分割：CNN编码器→解码器结构
- 生成对抗网络：生成器中的特征图上采样
- 目标检测：FPN等特征金字塔结构

**实现方式**：
- 1×1padding，stride 2的反卷积
- stride 1，padding的反卷积
- 双线性插值（有时更简单有效）

---

### Q14: 什么是感受野？如何计算？

**Answer:**

**感受野（Receptive Field）**：特征在输入图像上对应的原始像素区域

**计算公式**：
```
j_out = j_in × stride
r_out = r_in + (kernel_size - 1) × j_in
```
其中：
- j = jump，相邻特征的距离
- r = receptive field，感受野大小

**示例**：
```
输入层: r=1, j=1
Conv 3×3, stride1: r=1+(3-1)×1=3, j=1
Conv 3×3, stride2: r=3+(3-1)×1=5, j=2
Conv 3×3, stride2: r=5+(3-1)×2=9, j=4
```

---

### Q15: 什么是池化层（Pooling）？

**Answer:**

**最大池化（Max Pooling）**：
```
Forward: 选择局部区域的最大值
Backward: 梯度只传给最大值位置
```
- 常用配置：2×2窗口，stride 2
- 输出尺寸减半

**平均池化（Average Pooling）**：
```
Forward: 计算局部区域的平均值
Backward: 梯度均分到所有位置
```

**全局平均池化（Global Average Pooling）**：
- 将每个通道的空间信息压缩为单一值
- 替代全连接层，减少参数量
- 常用作分类网络的最后一步

## 第三部分：优化算法

---

### Q16: 什么是随机梯度下降（SGD）？

**Answer:**

**Mini-batch SGD**：
```
θ = θ - η · ∇J(θ; B)
```
其中B是一个mini-batch的训练样本

**为什么用mini-batch**：
- 计算整个训练集的梯度太慢
- batch size建议：8-16（用BN层时）
- 较大batch能利用GPU并行计算

**挑战**：
- 梯度估计有噪声
- 可能被困在局部最优
- 曲面很复杂时收敛慢

---

### Q17: 什么是Momentum？为什么有用？

**Answer:**

**动量更新**：
```
Δθ(t) = momentum · Δθ(t-1) + (1-momentum) · ∇Jθ(θ(t))
θ = θ - η · Δθ(t)
```

**直观理解**：
- 积累历史梯度信息
- 减少在峡谷壁之间的振荡
- 在梯度一致的方向上加速

**物理类比**：
- 就像一个球在曲面上滚动
- 动量帮助球越过小坑继续前进
- 参数momentum通常设为0.9

---

### Q18: 什么是Adam优化器？

**Answer:**

**Adam = Adaptive Moment Estimation**

**核心思想**：为每个参数自适应调整学习率

**算法**：
```
m(t) = β1 · m(t-1) + (1-β1) · g(t)        # 梯度的一阶矩估计
v(t) = β2 · v(t-1) + (1-β2) · g(t)²      # 梯度的二阶矩估计

m̂ = m(t) / (1-β1^t)                        # 偏差校正
v̂ = v(t) / (1-β2^t)

θ = θ - η · m̂ / (√v̂ + ε)                  # 更新
```

**默认参数**：
- β1 = 0.9
- β2 = 0.999
- ε = 10^-8

---

### Q19: 什么是AdamW？

**Answer:**

**问题**：原始Adam中L2正则化会与自适应学习率产生干扰

**AdamW解决方案**：
```
θ = θ - η · (m̂/(√v̂+ε) + λ·θ)
```
将权重衰减解耦（decoupled）开来

**对比**：
| 方法 | 更新方式 |
|------|---------|
| Adam + L2 | 梯度中包含λ·θ项 |
| AdamW | 权重衰减独立于梯度估计 |

**实际应用**：大多数现代深度学习框架默认使用AdamW

---

### Q20: 什么是学习率调度（Learning Rate Schedule）？

**Answer:**

**阶梯衰减（Step Decay）**：
```
lr = lr × 0.1  当 #iteration % #step == 0
```
- AlexNet、VGG等使用

**指数衰减（Exponential Decay）**：
```
lr = lr(0) × exp(-k × #iteration)
```
- GoogLeNet使用

**余弦退火（Cosine Annealing）**：
```
lr(t) = lr_min + (lr_max - lr_min) × (1 + cos(π×t/T)) / 2
```

**Warmup**：
- 训练初期逐步增大学习率
- 对Transformer等架构特别重要
- 防止早期梯度不稳定

## 第四部分：训练技术

---

### Q21: 什么是梯度消失和梯度爆炸？

**Answer:**

**梯度消失（Vanishing Gradients）**：
- 底层梯度变得非常小，接近零
- 原因：多层小于1的值相乘
- 多见于Sigmoid/Tanh激活函数
- 影响：网络几乎无法训练

**梯度爆炸（Exploding Gradients）**：
- 底层梯度变得非常大
- 原因：多层大于1的值相乘
- 影响：训练不稳定，参数更新过大

**解决方案**：
| 方法 | 说明 |
|------|------|
| Batch Normalization | 标准化层间激活值 |
| 残差连接 | 提供梯度的直接通路 |
| 好的初始化 | Xavier/Kaiming初始化 |
| 梯度裁剪 | 限制梯度范数 |

---

### Q22: 如何初始化权重？

**Answer:**

**问题**：初始化不当会导致梯度消失/爆炸

**Xavier初始化**（适用于Tanh/Sigmoid）：
```
W ~ N(0, 2/(d_in + d_out))
W ~ U(-√6/(√(d_in+d_out)), √6/(√(d_in+d_out)))
```

**Kaiming初始化**（适用于ReLU）：
```
W ~ N(0, 2/d_in)
```
考虑ReLU将一半神经元置零的特性

**关键洞察**：
- 保持前向传播时方差一致
- 保持反向传播时梯度方差一致

---

### Q23: 什么是数据增强（Data Augmentation）？

**Answer:**

**目的**：通过变换现有样本增加训练数据多样性

**图像分类常用技术**：

| 方法 | 说明 |
|------|------|
| 水平翻转 | 最常用，几乎所有任务 |
| 随机裁剪 | Random Crop/Scale |
| 颜色抖动 | 随机改变亮度、对比度 |
| PCA颜色偏移 |沿主成分方向偏移颜色 |
| 旋转/缩放 | 轻度变换 |
| 添加噪声 | 提高鲁棒性 |

**测试时增强（TTA）**：
- 多个增强版本的预测结果平均
- 通常能提升2-3%准确率

---

### Q24: 什么是多GPU训练？

**Answer:**

**为什么需要多GPU**：
- 单GPU显存有限（12-32GB）
- 训练大模型需要更大显存
- 加速训练过程

**数据并行（Data Parallelism）**：
- 模型在每个GPU上有完整副本
- mini-batch分割到多个GPU
- 梯度同步后更新参数
- 通信量大，但实现简单

**模型并行（Model Parallelism）**：
- 模型拆分到多个GPU
- 每个GPU只负责部分计算
- 适合超大模型
- 通信模式复杂

**混合并行**：
- 数据并行+模型并行
- 大规模训练常用策略

## 🎯 重点难点解析

---

### Q25: 反向传播的链式法则如何应用？

**Answer:**

**核心思想**：从输出层到输入层，逐层计算梯度

**示例（全连接层）**：
```
Forward: x → [FC] → y → [Softmax] → p → [CrossEntropy] → J
```

**Backward**：
```
∂J/∂p → ∂p/∂y → ∂y/∂W → ∂J/∂W
∂J/∂p → ∂p/∂y → ∂y/∂x → ∂J/∂x
```

**计算图视角**：
- 每个节点：前向输入z，输出a，局部梯度∂output/∂z
- 反向传播：接收上游梯度∂J/∂output，返回∂J/∂input

---

### Q26: CNN的反向传播有什么特点？

**Answer:**

**卷积层梯度计算**：
```
∂J/∂W = rot180(∂J/∂y) * x
∂J/∂x = rot180(W) * ∂J/∂y
```
- 使用**互相关（cross-correlation）**而非卷积
- 需要对核进行180度旋转

**池化层梯度**：
- Max Pooling：梯度只传到最大值位置
- Average Pooling：梯度均分到所有位置

---

### Q27: 为什么ResNet能训练很深的网络？

**Answer:**

**核心创新**：残差连接（Shortcut Connection）
```
y = F(x) + x
```

**梯度流优势**：
- 梯度可以直接流向浅层
- 缓解深层网络的梯度消失问题
- 即使F(x)学习不到任何东西，至少可以学习恒等映射

**数学分析**：
```
∂J/∂x = ∂J/∂y · ∂y/∂x
      = ∂J/∂y · (∂F/∂x + 1)
```
即使∂F/∂x很小，还有"+1"保证梯度传播
