## 📖 Lecture 1: MapReduce 基础

### Q: 什么是 MapReduce？

MapReduce 是第一个广泛使用的分布式计算编程模型，用于表达大规模分布式计算。

核心思想：**分治法 (Divide and Conquer)**

```
输入数据 → Map → 分区 → Shuffle & Sort → Reduce → 输出结果
```

### Q: MapReduce 的核心组件有哪些？

| 组件 | 说明 |
|------|------|
| **Map** | 将输入数据转换为 key-value 对 |
| **Reduce** | 将相同 key 的所有 value 聚合 |
| **Partitioner** | 决定哪个 reducer 处理哪个 key |
| **Combiner** | 在 map 端进行本地聚合，减少网络传输 |

### Q: MapReduce 的执行流程是怎样的？

1. **输入分片**：大数据被分成多个 splits
2. **Map 阶段**：每个 worker 处理一个 split，输出中间 key-value 对
3. **Shuffle 阶段**：根据 key 将数据重新分区，传输到对应的 reducer
4. **Reduce 阶段**：对每个 key 的所有 value 进行聚合
5. **输出**：结果写入 HDFS

### Q: Hadoop 1.0 和 Hadoop 2.0 (YARN) 的区别？

| Hadoop 1.0 | Hadoop 2.0 (YARN) |
|------------|-------------------|
| JobTracker 单点故障 | ResourceManager 分布式管理 |
| 只支持 MapReduce | 支持多种计算框架 |
| 资源分配不灵活 | 资源管理更灵活 |

### Q: MapReduce 性能优化策略？

1. **数据本地化**：将计算任务调度到数据所在的节点，减少网络传输
2. **Combiner**：在Map端进行本地聚合，减少网络传输量
3. **分区优化**：合理设计分区函数，确保数据均匀分布
4. **压缩**：对中间数据和输出数据进行压缩，减少I/O和网络传输
5. **内存优化**：调整Map和Reduce任务的内存分配
6. **并行度调整**：根据集群规模和数据量调整Map和Reduce任务数

### Q: 云计算安全机制？

1. **身份认证**：使用Kerberos等机制进行身份验证
2. **授权管理**：基于角色的访问控制（RBAC）
3. **数据加密**：对静态数据和传输中数据进行加密
4. **网络安全**：使用VPC、防火墙等隔离网络环境
5. **审计日志**：记录所有操作，便于安全审计和故障排查
6. **漏洞管理**：定期扫描和修复系统漏洞

## 📖 Lecture 2: 频繁项挖掘 (Frequent Itemsets)

### Q: 什么是频繁项挖掘？

频繁项挖掘是**关联规则发现**的核心技术，用于找出一起出现的物品组合。

经典案例：**尿布与啤酒**
- 规则：{尿布} → {啤酒}
- 解释：购买尿布的父亲通常也会购买啤酒

### Q: 频繁项挖掘的基本概念？

| 概念 | 说明 |
|------|------|
| **支持度 (Support)** | 物品组合出现在多少个篮子中 |
| **置信度 (Confidence)** | 购买 A 的人中，有多少也购买 B |
| **频繁项集** | 支持度大于阈值的项集 |

### Q: A-Priori 算法原理？

A-Priori 算法利用**先验性质**：
> 如果一个项集是频繁的，那么它的所有子集也必须是频繁的

**两阶段方法**：
1. 第一遍：找出所有频繁单项
2. 第二遍：只计算候选配对（由两个频繁单项组成）

### Q: PCY 算法是什么？

PCY (Park-Chen-Yu) 算法是对 A-Priori 的改进：
- 第一遍使用**哈希表**将项对映射到桶
- 只对哈希到频繁桶的项对进行计数
- 大大减少内存需求 |

## 📖 Lecture 3: 相似项查找 (Similar Items - LSH)

### Q: 为什么要研究相似项查找？

在大数据环境下，很多问题可以归结为找"相似"的集合：
- 找出相似的网页（去重检测）
- 找出购买相似商品的客户
- 找出相似特征的图片
- 找出访问相似网站的用户

### Q: MinHash 是什么？

MinHash 是用于估计 Jaccard 相似度的技术：

**Jaccard 相似度** = |A ∩ B| / |A ∪ B|

MinHash 的核心思想：对集合进行哈希，相同元素哈希值相等的概率等于 Jaccard 相似度。

### Q: Locality Sensitive Hashing (LSH) 是什么？

LSH 是用于**快速近似最近邻搜索**的技术：

- 将相似项映射到相同的"桶"
- 不相似的项映射到相同桶的概率很低
- 常用方法：**MinHash LSH**、**Euclidean LSH**

### Q: LSH 的应用场景？

- **网页去重**：检测相似或重复的网页
- **图像检索**：找相似的图片
- **推荐系统**：找相似的用户/物品
- **文档查重**：检测抄袭

## 📖 Lecture 4: 聚类 (Clustering)

### Q: 什么是聚类？

聚类是将数据点分组的技术，使得：
- 同一簇内的点**相似**
- 不同簇的点**不相似**

### Q: 常见的聚类算法？

| 算法 | 说明 | 优点 |
|------|------|------|
| **K-Means** | 迭代更新质心 | 简单高效 |
| **GMM (高斯混合模型)** | 概率模型 | 软聚类 |
| **层次聚类** | 构建层次树 | 不需要预设 K |

### Q: K-Means 算法的步骤？

1. 随机选择 K 个质心
2. 将每个点分配给最近的质心
3. 更新质心为簇内点的均值
4. 重复步骤 2-3 直到收敛

### Q: 聚类的挑战？

- **维度灾难**：高维空间中距离度量失效
- **初始化敏感**：不同初始化导致不同结果
- **难以确定 K**：需要预先指定簇数量

## 📖 Lecture 5: Spark 基础

### Q: 为什么要学习 Spark？

传统 MapReduce 的问题：
- 每次迭代都需要读写磁盘
- 不适合迭代算法（机器学习）
- 延迟高，不适合交互式分析

**Spark 优势**：
- **内存计算**：数据缓存在内存中
- **DAG 执行引擎**：优化执行计划
- **统一栈**：批处理、流处理、交互查询

### Q: Spark 的核心概念？

| 概念 | 说明 |
|------|------|
| **RDD** | 弹性分布式数据集 |
| **DataFrame** | 表格型数据抽象 |
| **DataSet** | 强类型的数据抽象 |
| **Transformation** | 转换操作（惰性） |
| **Action** | 触发执行 |

### Q: Spark vs MapReduce？

| 特性 | MapReduce | Spark |
|------|-----------|-------|
| 数据存储 | HDFS | 内存 + HDFS |
| 迭代运算 | 慢 | 快 |
| 编程模型 | MR | RDD/DataFrame |
| 容错 | 重算 | Lineage |

### Q: Spark 性能优化策略？

1. **缓存策略**：使用cache()或persist()缓存频繁使用的RDD
2. **数据序列化**：选择合适的序列化方式（如Kryo）减少内存使用
3. **分区优化**：合理设置分区数，避免数据倾斜
4. **广播变量**：使用广播变量减少数据传输
5. **累加器**：使用累加器进行高效的全局计数
6. **内存管理**：调整executor内存分配，优化内存使用
7. **任务并行度**：根据集群资源调整并行度
8. **数据本地性**：优化任务调度，提高数据本地性

### Q: 为什么要学习 Pig 和 Hive？

虽然 MapReduce 强大，但编写 Java 代码太繁琐。Pig 和 Hive 提供了高级语言：

| 工具 | 语言 | 开发者 |
|------|------|--------|
| **Pig** | Pig Latin (类似 Perl) | Yahoo! |

### Q: Pig 的特点？

- 用 Pig Latin 编写数据流脚本
- 2009 年 Yahoo! 约 40% 的 Hadoop 作业使用 Pig
- 高级语言"编译"成 MapReduce 作业

### Q: Hive 的特点？

- 类似 SQL 的查询语言
- 表存储在 HDFS 作为平面文件
- 适合数据分析人员（不需要写 Java）

### Q: Pig vs Hive？

- Pig：适合数据流水线处理
- Hive：适合熟悉 SQL 的用户

## 📖 Lecture 8: 推荐系统 (Recommender Systems)

### Q: 什么是推荐系统？

推荐系统是**预测用户可能感兴趣的内容**的技术，广泛应用于：
- 电子商务（亚马逊、淘宝）
- 流媒体（Netflix、YouTube）
- 社交网络（Facebook、Twitter）

### Q: 推荐系统的核心问题？

| 问题 | 说明 |
|------|------|
| **评分预测** | 预测用户对物品的评分 |
| **Top-N 推荐** | 为用户推荐 N 个最相关的物品 |
| **协同过滤** | 利用用户行为数据进行推荐 |

### Q: 协同过滤的核心思想？

> "相似用户会喜欢相似的物品"

- **基于用户的协同过滤**：找相似用户，推荐他们喜欢的物品
- **基于物品的协同过滤**：找相似物品，推荐给用户

### Q: 矩阵分解是什么？

矩阵分解是**将用户-物品评分矩阵分解为两个低维矩阵**：

```
R ≈ U × V^T
```

- U：用户隐因子矩阵
- V：物品隐因子矩阵
- 常用方法：SVD、ALS

### Q: 推荐系统的挑战？

- **冷启动问题**：新用户/新物品缺乏历史数据
- **稀疏性**：用户评分的物品远少于总物品数
- **可扩展性**：海量用户和物品的处理
- **实时性**：实时推荐的需求

## 📖 Lecture 9: 维度约简 (Dimensionality Reduction)

### Q: 什么是维度约简？

维度约简是将高维数据压缩到低维空间的技术，同时保留数据的关键信息。

### Q: 为什么要做维度约简？

- **发现隐藏关联**：找出属性间的隐藏关系
- **去噪**：移除冗余和噪声特征
- **可视化**：将高维数据投影到 2D/3D
- **降低存储和计算成本**

### Q: 常见的方法有哪些？

| 方法 | 说明 |
|------|------|
| **SVD (奇异值分解)** | 矩阵分解，保留主要信息 |
| **PCA (主成分分析)** | 找出方差最大的正交方向 |

### Q: SVD 和 PCA 的关系？

SVD 是 PCA 的一种实现方式。PCA 找的是数据方差最大的方向，SVD 通过对数据矩阵进行分解实现类似效果。

## 📖 Lecture 10: Hadoop 1.0 vs 2.0 资源管理

### Q: Hadoop 1.0 有什么局限性？

| 问题 | 说明 |
|------|------|
| **可扩展性限制** | 最大 4000 节点，40000 并发任务 |
| **单点故障** | JobTracker 失败导致所有作业终止 |
| **仅支持 MapReduce** | 不适合其他计算模型 |
| **资源分配不灵活** | 资源利用率低 |

### Q: YARN 是什么？

YARN (Yet Another Resource Negotiator) 是 Hadoop 2.0 的资源管理框架：

- **ResourceManager**：管理整个集群资源
- **NodeManager**：管理单个节点的资源
- **ApplicationMaster**：为每个应用请求资源

### Q: YARN 的优势？

- 支持多种计算框架（Spark、Flink 等）
- 更高的资源利用率
- 更好的可扩展性

- **问题分析**：交通数据量大，实时性要求高
- **解决方案**：
  - 使用Kafka收集实时交通数据
  - 用Flink进行流处理和分析
  - 构建交通预测模型，优化信号灯控制
- **效果**：交通拥堵减少30%，平均通行时间缩短25%
