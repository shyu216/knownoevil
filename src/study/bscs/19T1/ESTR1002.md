## Q&A 形式课程详解

### Chapter: Arrays and Loops 数组与循环

---

### Q1: 什么是数组？为什么需要数组？

**Answer:**

数组（Array）是一种 fundamental data structure，用于更好地组织数据。

**举例说明：**

假设你需要存储 50 名学生的成绩：

| 不使用数组 | 使用数组 |
|-----------|---------|
| `int score1, score2, score3, ... score50;` | `int scores[50];` |
| 麻烦！需要定义50个变量 | 优雅！一个数组搞定 |

数组就像一个带有很多抽屉的柜子，每个抽屉（元素）存储一个值。

---

### Q2: 数组的基本特性有哪些？

**Answer:**

1. **固定长度**：数组大小在创建时确定，整个生命周期内不能改变
2. **同类型元素**：所有元素必须是相同数据类型（homogeneous）
3. **索引访问**：通过索引（从0开始）快速访问任意元素

```
长度 = 8
索引    0     1     2     3     4     5     6     7
元素    5     6     8     5     6     8     7     1
```

---

### Q3: 如何声明和初始化数组？

**Answer:**

```c
// 声明一个包含5个整数的数组
int score_array[5];

// 初始化所有元素为0
int score_array[5] = {0, 0, 0, 0, 0};

// 推断数组大小（自动设为4）
int score_array[] = {1, 2, 3, 4};

// 部分初始化，其余自动设为0
int score_array[5] = {1, 1};  // 结果: {1, 1, 0, 0, 0}
```

**⚠️ 重要注意事项：**
- 数组大小在**编译时**必须已知
- 数组大小一旦确定，**不能改变**

---

### Q4: 如何访问和遍历数组元素？

**Answer:**

数组索引**从0开始**，到 `length - 1` 结束：

```c
int score[5];

score[0] = 0;    // 第一个元素
score[1] = 1;    // 第二个元素
score[4] = 100;  // 最后一个元素（索引为4）

printf("%d\n", score[0]);  // 读取第一个元素
```

**通常结合 for 循环遍历数组：**

```c
int score[100];
int i;

// 给所有元素赋值
for (i = 0; i < 100; i++) {
    score[i] = i;
}

// 读取所有元素
for (i = 0; i < 100; i++) {
    printf("%d\n", score[i]);
}
```

---

### Q5: 使用数组时有哪些常见错误？

**Answer:**

| 错误类型 | 示例 | 后果 |
|---------|------|------|
| **数组越界** | `score[5] = 0;`（数组大小为5） | 程序可能崩溃 |
| **未初始化** | `int score[100]; printf("%d", score[0]);` | 值是随机的 |
| **运行时决定大小** | `scanf("%d", &n); int array[n];` | 部分编译器不支持 |

> **必须**在使用前给数组元素赋初值！数组越界可能导致 Segmentation Fault！

---

### Q6: 数组可以作为查找表使用吗？

**Answer:**

当然可以！这是一个非常实用的技巧：

```c
// 月份转天数（平年）
int monthToDays[12] = {31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};

int month;
scanf("%d", &month);
int days = monthToDays[month - 1];
printf("%d月有%d天\n", month, days);
```

**优势**：无需使用 if-else 或 switch 语句，直接通过索引查找！

### Q7: 如何正确拷贝和比较数组？

**Answer:**

**❌ 错误的做法：**
```c
// 拷贝 - 错误！
destination = source;  // 这只是复制指针

// 比较 - 错误！
if (array1 == array2)  // 这只比较指针地址
```

**✅ 正确的做法：**

```c
// 正确拷贝：逐元素拷贝
for (i = 0; i < 3; i++) {
    destination[i] = source[i];
}

// 正确比较：逐元素比较
int same = 1;  // 先假设相同
for (i = 0; i < 3; i++) {
    if (array1[i] != array2[i]) {
        same = 0;
        break;
    }
}
```

### Q8: 什么是二维数组？如何使用？

**Answer:**

二维数组本质上是"数组的数组"：

```c
int a[3][4];  // 3行4列
```

```
       列
       0    1    2    3
行 0  a[0][0] a[0][1] a[0][2] a[0][3]
    1  a[1][0] a[1][1] a[1][2] a[1][3]
    2  a[2][0] a[2][1] a[2][2] a[2][3]
```

**遍历二维数组（使用嵌套循环）：**
```c
int a[4][3];
int i, j;

for (i = 0; i < 4; i++) {
    for (j = 0; j < 3; j++) {
        a[i][j] = 1;
    }
}
```

### Q9: 如何判断一个数是否为质数？

**Answer:**

**暴力方法（低效）：**
```c
for (i = 2; i < input; i++) {
    if (input % i == 0) break;
}
if (i == input) printf("YES");  // 是质数
```

**优化方法（高效）：**
```c
for (i = 2; i <= input / i; i++) {
    if (input % i == 0) break;
}
if (i > input / i) printf("YES");  // 是质数
```

**效率对比：**
- 暴力方法：约 N 次循环
- 优化方法：约 √N 次循环
- 对于输入 4294967291：暴力方法需要 **10.1秒**，优化方法**几乎瞬间**完成！

### Q10: 什么是埃拉托斯特尼筛法？

**Answer:**

Sieve of Eratosthenes 是一种高效找出范围内所有质数的算法，由古希腊数学家 Eratosthenes 于公元前240年发明。

**算法步骤：**
1. 假设所有数都是质数（设为1）
2. 从2开始，对每个质数，将其倍数标记为非质数（设为0）
3. 重复直到 √N
4. 剩余的质数就是真正的质数

**代码实现：**
```c
int i, j, sqrtN, counter = 0;
char sieve[5000000 + 1];

// 初始化
for (i = 0; i <= 5000000; i++) {
    sieve[i] = 1;
}

// 筛选
sqrtN = (int)(sqrt(5000000) + 0.5);
for (i = 2; i <= sqrtN; i++) {
    if (sieve[i] == 1) {
        for (j = i + i; j <= 5000000; j += i) {
            sieve[j] = 0;
        }
    }
}

// 统计结果
for (i = 2; i <= 5000000; i++) {
    if (sieve[i] == 1) counter++;
}
```

**性能**：在 5,000,000 范围内找质数，仅需 **0.047秒**！

## 🎯 Key Points 重点总结

| 概念 | 要点 |
|------|------|
| **数组定义** | 固定大小的同类型数据集合 |
| **索引** | 从 0 开始 |

## 💡 Study Tips 学习建议

1. **多练习数组操作**：特别是与循环的结合
2. **注意索引边界**：始终检查 `i < length` 而不是 `i <= length`
3. **理解指针概念**：数组名本质上是指向首元素的指针（后续课程会详细讲）
4. **调试时打印索引**：帮助理解数组访问

## ⚠️ Common Pitfalls 常见陷阱

1. `array[index]` 中 index 越界
2. 忘记初始化数组
3. 使用 `=` 而不是 `==` 比较
4. 循环条件多写或少写一次（`i <=` vs `i <`）
