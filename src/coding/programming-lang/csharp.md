C# 是 2000 年微软发布的，当时就想给企业应用搞一门现代化的面向对象编程语言，构建在 .NET 平台之上，现在可是企业级应用开发的主流选手。

比如说，这三个用 C# 实现的项目：1. **Unity** - 开源游戏引擎，用于开发手游如《原神》《王者荣耀》，跨平台能力强、社区活跃，不过2019年后商业化条款收紧、授权费用上涨；2. **ASP.NET Core** - 开源Web框架，性能优秀、跨平台，不过生态系统不如Java丰富；3. **ReSharper** - 闭源Visual Studio插件，代码分析强大、效率提升显著，不过收费昂贵、影响IDE性能。

架构师选 C# 的时候，心里肯定在想："这玩意儿有 LINQ 查询数据超方便，async/await 写异步代码跟喝水一样简单，自动垃圾回收不用操心内存，跨平台支持越来越强，Windows、Linux、macOS 都能跑，强类型系统编译时就能抓错，面向对象特性齐全。虽然要了解 .NET 生态系统有点麻烦，早期版本只能跑在 Windows 上（现在好了），语法比一些现代语言冗长，.NET 框架体系庞大要花时间学，但架不住它好用啊！" 所以在企业级 Web 应用、桌面应用、移动应用、游戏开发（Unity）、云服务（Azure）、Windows 平台应用这些地方，C# 简直是如鱼得水。

和 C# 相关的语言有 Java，同为企业级应用开发的主流选手；Go 适合高并发后端服务；Python 在数据科学和脚本领域很厉害；F# 是 .NET 生态系统中的函数式编程语言。但 C# 就是在微软生态和 Windows 平台上特别能打，有微软持续投入，企业级应用开发选它准没错。

不过学 C# 也有难点，要了解 .NET 生态系统，依赖这个运行环境；早期版本仅限于 Windows，虽然现在改善了；语法相对冗长，比一些现代语言繁琐；.NET 框架体系庞大，需要时间掌握。

总的来说，C# 就像一个全能的企业级瑞士军刀，功能强大、生态丰富，特别是在 Windows 平台和云服务领域，简直是如虎添翼，是企业级应用开发的主流选择。

### 一个代码块

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello " + "Dale" + "!");
    }
}
```

运行只需要 `dotnet run`，需要先创建一个项目 `dotnet new console -o hello`。
