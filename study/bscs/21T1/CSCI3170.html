CSCI3170 Database Systems 数据库系统

Dale2025/5/11大约 12 分钟CUHKCourse21T1Computer ScienceDatabaseSQL

课程简介 (What is this course about?)

CSCI3170 - 数据库系统 是计算机科学的核心课程，探讨如何有效地存储、管理和检索大规模数据。数据库系统是现代信息系统的基石，从银行交易到社交网络，无处不在。

课程定位：

  • 面向计算机科学与工程专业学生
  • 数据库理论与实践的核心课程
  • 为后端开发、数据工程、大数据技术打下基础

知识在计算机大厦中的位置：

┌─────────────────────────────────────┐
│     大数据、数据仓库、数据挖掘        │
├─────────────────────────────────────┤
│  ← CSCI3170: 数据库系统 →           │
├─────────────────────────────────────┤

└─────────────────────────────────────┘

核心内容：

  • ER模型：实体-关系建模
  • 关系模型：表格表示、约束条件
  • 关系代数：查询操作的理论基础
  • SQL：结构化查询语言

未来发展方向：

  • NoSQL数据库：键值存储、文档数据库、图数据库
  • 分布式数据库：CAP定理、数据分片、复制
  • 数据库内核优化：索引技术、查询优化
  • 时序数据库与向量数据库：AI时代的新需求

📚 课程笔记 (Course Notes)

第一部分：数据库基础概念

Q1: 什么是数据库？它和文件系统有什么区别？

Answer:

数据库（Database） 是有组织的数据集合，通常描述一个或多个相关组织的活动。它模型化现实世界的实体（如学生、教授、课程）和实体之间的关系（如学生选课）。

文件系统的问题：

  • 应用必须处理大量主存和辅存之间的数据缓冲
  • 页面级访问、地址处理复杂
  • 回答不同查询需要专门的程序
  • 难以处理多用户并发访问的一致性问题
  • 缺乏崩溃恢复机制
  • 安全性和访问控制薄弱

DBMS的优势：

  • 数据独立性：应用不暴露数据表示和存储的细节
  • 高效数据访问：支持复杂的存储和检索技术
  • 数据完整性和安全性：强制完整性约束和访问控制
  • 并发控制和崩溃恢复：保护用户免受系统故障影响
  • 减少应用开发时间：提供高层接口

Q2: 数据库的三层抽象是什么？

Answer:

层次描述说明
物理层（Physical Level）描述数据如何以及在哪里实际存储指定底层数据结构和文件组织、索引等
概念层（Conceptual Level）描述数据库中存储什么数据及数据之间的关系和语义使用数据模型（如关系模型）描述
视图层（View Level）特定用户看到的数据的部分视图由一个或多个视图和概念模式的部分组成

数据独立性：

  • 物理数据独立性：改变物理数据库模式而不改变概念模式
  • 逻辑数据独立性：改变概念模式而不影响外部视图

Q3: 数据模型是什么？有哪些类型？

Answer:

数据模型 是用于描述数据、数据关系和数据约束的概念集合。

语义数据模型（高级抽象）：

  • 更抽象、高层次的数据描述
  • 例子：实体-关系（ER）模型
  • 用图形方式表示实体及实体间的关系

关系数据模型（当今主流）：

  • 关系（Relation） 是核心数据结构
  • 可以看作记录的集合，即带行列的表格
  • 行（Row）= 记录（Tuple）
  • 列（Column）= 字段/属性（Field/Attribute）

第二部分：ER模型（Entity-Relationship Model）

Q4: 什么是实体和属性？

Answer:

实体（Entity）：现实世界中可区分的对象

  • 例如：一位老师、一间教室
  • 由一组**属性（Attributes）**描述

属性类型：

类型说明例子
简单属性不可再分年龄、性别
复合属性可再分为多个部分地址（省、市、区）
单值属性每个实体只有一个值生日
多值属性一个实体有多个值电话号码
派生属性可从其他属性计算得出年龄（从生日计算）

超键（Superkey）：能唯一标识实体的属性集合
键/候选键（Key/Candidate Key）：最小化的超键
主键（Primary Key）：从候选键中选择的一个

Q5: 什么是关系？有哪些类型？

Answer:

关系（Relationship）：两个或多个实体之间的关联

关系的度（Degree）：

  • 二元关系（Binary）：两个实体集之间
  • 三元关系（Ternary）：三个实体集之间
  • N元关系（N-ary）：N个实体集之间

映射基数（Mapping Cardinality）：

类型描述例子
一对一（1:1）A中每个实体至多关联B中一个实体，反之亦然经理 ↔ 部门
一对多（1:N）B中每个实体可关联A中多个实体，但A中每个实体至多关联B中一个母亲 ↔ 孩子
多对一（N:1）一对多的反向员工 ↔ 部门
多对多（M:N）无限制学生 ↔ 课程

Q6: 什么是参与约束和弱实体？

Answer:

参与约束（Participation Constraints）：

  • 完全参与（Total Participation）：每个实体必须参与至少一个关系（用双线表示）
  • 部分参与（Partial Participation）：实体可能不参与任何关系

弱实体集（Weak Entity Set）：

  • 没有主键的实体集
  • 只能通过与另一个实体集（所有者）的关联来唯一标识
  • 部分键（Partial Key）：在给定所有者的情况下，唯一标识弱实体的属性集合

弱实体的识别：

  • 依赖于另一个实体集（Identifying Owner）
  • 通过所有者实体键和部分键的组合来标识

Q7: 什么是ISA层次结构和聚合？

Answer:

ISA层次结构（Class Hierarchies）：

  • 将实体集自然地分类为子类
  • 属性继承：子类继承父类的所有属性
  • 重叠约束：两个子类是否允许包含相同实体
  • 覆盖约束：子类是否共同包含父类中的所有实体

聚合（Aggregation）：

  • 用于建模关系与实体之间的关系
  • 将一个关系集（通过虚线框标识）参与另一个关系集
  • 例如：员工监控项目赞助关系

第三部分：关系模型（Relational Model）

Q8: 关系模型的基本概念是什么？

Answer:

历史：由Codd于1970年提出，今天已成为最主流的数据模型。

关系结构：

  • 关系模式（Relation Schema）：描述表的列，指定关系名、每个字段名及其域
  • 关系实例（Relation Instance）：表的实际数据，是元组的集合

关系特征：

  • 不允许有完全相同的两行
  • 行的顺序不重要
  • 列的顺序不重要
  • 每个列的值必须来自该列的域（域约束）

关键术语：

  • 度（Degree）：字段的数量
  • 基数（Cardinality）：元组的数量
  • 候选键（Candidate Key）：唯一标识元组的最小属性集
  • 主键（Primary Key）：从候选键中选择

Q9: 什么是外键约束？

Answer:

外键（Foreign Key）：

  • 一个关系中引用另一个关系主键的属性集
  • 必须对应到被引用关系中存在的值
  • 确保参照完整性

参照完整性约束：外键值必须匹配被引用关系的主键值

违反处理方式：

选项说明
NO ACTION（默认）拒绝删除/更新操作
CASCADE级联删除/更新相关记录
SET NULL将外键设置为NULL
SET DEFAULT设置为默认值

第四部分：关系代数（Relational Algebra）

Q10: 关系代数的基本运算符有哪些？

Answer:

六大基本运算符：

运算符符号作用
选择（Selection）σ选择满足条件的元组
投影（Projection）Π提取指定列
并（Union）∪合并两个关系的元组
差（Set Difference）-返回在R中但不在S中的元组
笛卡尔积（Cartesian Product）×合并两个关系的所有组合
重命名（Rename）ρ重命名关系或属性

选择（σ）：

σ condition (R)
  • 条件是布尔表达式，使用 ∧ (AND)、∨ (OR)、¬ (NOT)
  • 例：σ dno=4 ∧ salary>25000 (Employee)

投影（Π）：

Π attribute-list (R)
  • 消除重复元组
  • 例：Π sname, rating (Sailors)

Q11: 什么是连接运算？

Answer:

连接（Join） 是笛卡尔积加选择操作的组合

条件连接（θ-Join）：

R ⋈cond S = σcond (R × S)

等值连接（Equi-Join）：

  • 连接条件只包含等式
  • 结果中会保留重复的属性

自然连接（Natural Join）：

  • 自动在所有同名列上等值连接
  • 结果中不保留重复列
  • 如果两关系无共同属性，则为笛卡尔积

示例：

Sailors ⋈ Reserves = SELECT * FROM Sailors, Reserves WHERE Sailors.sid = Reserves.sid

Q12: 什么是除法运算？

Answer:

除法（Division） 用于表达"对于每一个..."类型的查询

定义：

  • 给定关系A（属性x, y）和B（属性y）
  • A ÷ B 是所有x值的集合
  • 条件：对于B中的每个y值，元组<x, y>必须在A中

直观理解：

  • 对于A中的每个x值，考虑与该x值一起出现的y值集合
  • 如果这个集合包含B中的所有y值，则x在结果中

应用场景：

  • "找出预订了所有船只的水手"
  • "找出选修了所有课程的学生"

第五部分：SQL

Q13: SQL的基本查询格式是什么？

Answer:

基本SELECT语句：

SELECT [DISTINCT] target-list
FROM relation-list
WHERE qualification

三个子句的作用：

  • SELECT：指定要保留的列（投影）
  • FROM：指定交叉乘积的表
  • WHERE：指定选择条件（可选）

等价的关系代数：

Π target-list ( σ qualification (R1 × R2 × ... × Rm) )

执行策略（概念层面）：

  1. 计算FROM子句中表的交叉乘积
  2. 应用WHERE条件筛选
  3. 删除不在target-list中的属性
  4. 如果指定DISTINCT，消去重复行

Q14: SQL中的集合运算有哪些？

Answer:

三种集合运算：

运算SQL关键字说明
并集UNION返回两个查询结果的并集，自动去重
交集INTERSECT返回两个查询结果的交集
差集EXCEPT/MINUS返回在第一个结果中但不在第二个结果中的元组

重要约束：

  • 两个查询结果必须并集相容（相同的字段数量，对应字段类型相同）
  • 默认去除重复元组（可用ALL保留）

示例：

-- 预订了红船或绿船的水手sid
SELECT S.sid FROM Sailors S, Boats B, Reserves R
WHERE S.sid=R.sid AND R.bid=B.bid AND (B.color='red' OR B.color='green')
UNION
SELECT S.sid FROM Sailors S, Boats B, Reserves R
WHERE S.sid=R.sid AND R.bid=B.bid AND B.color='green'

Q15: 什么是嵌套查询？

Answer:

嵌套查询：在WHERE子句中嵌入另一个查询

关键操作符：

操作符说明
IN / NOT IN检查值是否在子查询结果中
EXISTS / NOT EXISTS检查子查询结果是否非空
UNIQUE检查子查询结果是否有重复
ANY / ALL与比较运算符一起使用

相关嵌套查询（Correlated Nested Queries）：

  • 内层查询依赖于外层查询的当前行
  • 对于外层查询的每个元组，内层查询都要重新计算

示例：

-- 找出预订了103号船的水手名字
SELECT S.sname FROM Sailors S
WHERE EXISTS (SELECT * FROM Reserves R
              WHERE R.bid=103 AND S.sid=R.sid)

Q16: SQL中的聚合函数有哪些？

Answer:

五大聚合函数：

函数说明
COUNT([DISTINCT] A)A列中（唯一）值的数量
SUM([DISTINCT] A)A列中（唯一）值的总和
AVG([DISTINCT] A)A列中（唯一）值的平均值
MAX(A)A列中的最大值
MIN(A)A列中的最小值

GROUP BY子句：

  • 按指定属性分组
  • 对每个组应用聚合函数

HAVING子句：

  • 对分组进行条件筛选
  • 条件必须对每个组产生单一值

示例：

-- 找出每个rating级别中年龄最小的水手
SELECT S.rating, MIN(S.age) FROM Sailors S
GROUP BY S.rating
HAVING COUNT(*) > 1

Q17: 什么是触发器？

Answer:

触发器（Trigger）：当数据库发生指定变化时自动执行的存储过程

三个组成部分：

  1. 事件（Event）：激活触发器的变化（如INSERT、UPDATE、DELETE）
  2. 条件（Condition）：测试触发器是否应该运行
  3. 动作（Action）：触发器运行时执行的操作

示例（SQL:1999）：

CREATE TRIGGER youngSailorUpdate
AFTER INSERT ON SAILORS
REFERENCING NEW TABLE NewSailors
FOR EACH STATEMENT
INSERT INTO YoungSailors(sid, name, age, rating)
SELECT sid, name, age, rating FROM NewSailors N WHERE N.age <= 18

🎯 重点难点解析

Q18: ER模型转换为关系模型的关键规则是什么？

Answer:

实体集 → 表：

  • 每个属性成为列
  • 主键成为主键约束

关系集 → 表：

  • 包含参与实体集的主键（作为外键）
  • 包含关系的描述性属性
  • 主键取决于映射约束

键约束的影响：

  • 多对多：主键是所有参与实体键的并集
  • 一对多/多对一：主键来自"多"端的实体键
  • 一对一：任一实体键都可以作为主键

弱实体集 → 表：

  • 包含弱实体自己的属性
  • 包含所有者实体集的主键（作为外键）
  • 主键是部分键和所有者主键的组合

Q19: 如何设计好的数据库模式？

Answer:

设计原则：

  1. 最小冗余：避免重复存储相同信息
  2. 完整性约束：使用主键、外键、CHECK约束确保数据正确性
  3. 规范化：消除插入、更新、删除异常

常见问题：

  • 数据冗余：重复存储导致存储浪费和更新困难
  • 更新异常：更新数据时可能造成不一致
  • 插入异常：无法插入某些缺少主键的数据
  • 删除异常：删除数据时可能意外丢失其他信息

解决方案：通过ER图进行概念设计，然后转换为关系模型

Q20: SQL中NULL值的三值逻辑是什么？

Answer:

NULL的三种含义：

  1. 值未知
  2. 值不适用
  3. 值 withheld（拒绝透露）

三值逻辑：

表达式结果
TRUE AND UNKNOWNUNKNOWN
FALSE AND UNKNOWNFALSE
UNKNOWN AND UNKNOWNUNKNOWN
TRUE OR UNKNOWNTRUE
FALSE OR UNKNOWNUNKNOWN
UNKNOWN OR UNKNOWNUNKNOWN
NOT UNKNOWNUNKNOWN

重要特性：

  • WHERE子句只保留结果为TRUE的行
  • COUNT(*) 包含NULL值
  • 其他聚合函数忽略NULL值
