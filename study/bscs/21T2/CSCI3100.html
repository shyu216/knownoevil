CSCI3100 Software Engineering 软件工程

Dale2025/5/11大约 10 分钟CUHKCourse21T2Computer ScienceSoftware EngineeringDevelopment

课程简介 (What is this course about?)

CSCI3100 - 软件工程 是香港中文大学计算机科学与工程系的软件工程入门课程，主要介绍软件工程的基本概念、原理和方法。软件工程是一门研究如何系统化、规范化、可量化地开发和维护软件的学科。

课程定位：

  • 面向计算机科学与工程专业学生
  • 软件开发的核心课程
  • 为项目管理、团队协作打下基础

知识在计算机大厦中的位置：

┌─────────────────────────────────────┐
│     项目管理、产品管理               │
├─────────────────────────────────────┤
│  ← CSCI3100: 软件工程 →              │
├─────────────────────────────────────┤
│     编程基础、数据结构                │
└─────────────────────────────────────┘

核心内容：

  • 软件生命周期：需求、设计、编码、测试、维护
  • 软件质量：可靠性、可维护性、可扩展性
  • 项目管理：敏捷开发、版本控制

未来发展方向：

  • 项目管理：Scrum、Kanban
  • DevOps：CI/CD、自动化测试
  • 架构设计：微服务、云原生

📚 课程笔记 (Course Notes)

Topic 1: Software Engineering: A Preview (软件工程概述)

核心内容：

  • 什么是软件工程：不仅仅是编程，而是处理复杂系统的系统性方法
  • 为什么需要软件工程：
    • 软件规模与成本每10年增长一个数量级
    • 应用程序复杂度增加，工程师生产力跟不上需求
    • 软件故障可能带来灾难性后果（如千年虫问题）
    • 软件演进和可维护性需求

软件生命周期（瀑布模型）：

  1. 需求分析与规格说明 (Requirements Analysis and Specification)
  2. 设计 (Design)
  3. 编码与单元测试 (Coding and Module Testing)
  4. 集成与系统测试 (Integration and System Testing)
  5. 交付与维护 (Delivery and Maintenance)

难点： 理解软件工程与普通编程的区别；掌握瀑布模型的各阶段及其验证点。

Topic 2: Software: Its Nature and Qualities (软件的本质与质量)

核心内容：

  • 软件特性：
    • 可变性：修改代码容易，但需要有纪律的变更管理
    • 人力密集型：每个阶段都需要人工介入
    • 质量难以定义：难以将产品与过程分离

软件质量分类：

  • 外部质量 vs 内部质量
  • 产品质量 vs 过程质量

代表性软件质量：

  1. 可靠性 (Reliability)：软件在规定环境下正常运行的概率
  2. 健壮性 (Robustness)：在未指定情况下的合理行为
  3. 可用性 (Availability)：MTBF / (MTBF + MTTR)
  4. 性能 (Performance)：效率与用户体验相关
  5. 可维护性 (Maintainability)：占总软件成本的60%以上！
    • 纠错性维护 (20%)
    • 适应性维护 (20%)
    • 完善性维护 (60%)
  6. 可重用性 (Reusability)：降低开发成本
  7. 可移植性 (Portability)：在不同环境运行
  8. 可理解性 (Understandability)：便于维护和验证

难点： 理解各种质量指标之间的权衡；理解维护成本的分布。

Topic 3: Software Engineering Principles (软件工程原则)

七大原则：

  1. 严格性与形式化 (Rigor and Formality)：用数学方法确保正确性
  2. 关注点分离 (Separation of Concerns)：将复杂问题分解处理
  3. 模块化 (Modularity)：分解为独立模块
  4. 抽象 (Abstraction)：忽略细节，关注本质
  5. 变更预判 (Anticipation of Change)：设计时考虑未来变化
  6. 通用性 (Generality)：设计通用解决方案
  7. 增量性 (Incrementality)：分步骤开发，快速原型

软件开发过程模型：

  • 瀑布模型 (Waterfall)
  • 极限编程 (XP - Extreme Programming)
  • 螺旋模型 (Spiral)

难点： 理解各原则之间的相互关系；在实际项目中应用这些原则。

Topic 4: Software Specification (软件规格说明)

规格说明的作用：

  • 明确用户需求
  • 定义接口
  • 指导设计过程
  • 维护参考点

规格说明质量要求：

  1. 清晰、无歧义
  2. 完整
  3. 可验证
  4. 可增量构建

规格说明分类：

  • 形式化 vs 非形式化
  • 操作性 vs 描述性
  • 数据范式 vs 过程范式

主要规格说明技术：

  1. 数据流图 (DFD)：描述数据在系统中的流动
  2. 有限状态机 (FSM)：描述系统状态转换
  3. Petri网：描述异步并发系统
  4. UML图表：
    • 用例图 (Use-case)
    • 类图 (Class)
    • 活动图 (Activity)
    • 序列图 (Sequence)
    • 协作图 (Collaboration)
  5. 实体-关系图 (ER)：描述数据模型
  6. 逻辑规格：使用一阶谓词逻辑

难点： FSM的状态爆炸问题；Petri网的死锁和饥饿分析；各种规格说明方法的适用场景。

Topic 5: Software Design (软件设计)

设计过程：

  • 将需求转化为软件"蓝图"
  • 从高层次抽象到详细设计

设计模型：

  1. 数据设计 (Data Design)
  2. 架构设计 (Architectural Design)
  3. 接口设计 (Interface Design)
  4. 过程/组件设计 (Procedural/Component Design)

模块关系：

  • USES关系：模块间的依赖关系，应为层次结构
  • IS_COMPONENT_OF关系：模块组成关系

关键概念：

  • 信息隐藏 (Information Hiding)：模块隐藏其内部实现细节
  • 抽象数据类型 (ADT)：通过接口隐藏数据表示
  • 内聚性 (Cohesion)：模块内部联系紧密程度
  • 耦合度 (Coupling)：模块间依赖程度

设计描述方法：

  • TDN (文本设计表示法)
  • GDN (图形设计表示法)

面向对象设计：

  • 继承 (Inheritance)
  • 多态 (Polymorphism)
  • 动态绑定 (Dynamic Binding)

UML在设计中的应用：

  • 类图
  • 继承表示
  • 关联 (Association)
  • 聚合 (Aggregation)
  • 组合 (Composition)

程序实现技术：

  • 编程标准与规范
  • 控制结构
  • 文档编写（内部+外部）

难点： 设计高效低耦合的模块结构；理解信息隐藏的重要性；掌握抽象数据类型的实现。

Topic 6: Verification: Testing and Analysis (验证：测试与分析)

验证 vs 确认：

  • 验证 (Verification)：是否正确构建产品？"Are we building the product right?"
  • 确认 (Validation)：是否构建正确的产品？"Are we building the right product?"

测试目标：

  • 发现bug而非证明无bug
  • 帮助定位错误
  • 测试应可重复

白盒测试（结构化测试）：

  1. 语句覆盖 (C0)：每个语句至少执行一次
  2. 边覆盖 (C1)：每条边至少遍历一次
  3. 条件覆盖 (C2)：每个条件的可能值都测试
  4. 路径覆盖 (C3)：所有路径都执行（指数级复杂度）

McCabe圈复杂度： V(G) = e - n + 2

黑盒测试（功能测试）：

  • 基于规格说明
  • 决策表测试
  • 因果图 (Cause-Effect Graph)
  • 边界值测试

测试组织：

  • 单元测试 (Unit Testing)
  • 集成测试 (Integration Testing)
  • 系统测试 (System Testing)

集成策略：

  • 自底向上 (Bottom-up)：从底层模块开始
  • 自顶向下 (Top-down)：从顶层模块开始
  • 三明治集成 (Sandwich)
  • 构建集成 (Build)

特殊测试：

  • 回归测试 (Regression Testing)
  • 性能测试
  • 健壮性测试

难点： 理解测试的局限性（无法证明无bug）；掌握各种覆盖标准的权衡；设计有效的测试策略。

评估方式

  • 作业 (60%)：

    • Assignment 1-3：涉及软件工程项目的不同阶段
    • 通常包括需求分析、设计、实现、测试文档

  • 期末考试 (40%)：涵盖所有主题

课程难点总结

  1. 软件质量权衡：理解各质量属性之间的相互影响和权衡
  2. 规格说明的形式化：掌握各种规格说明技术的适用场景
  3. 模块化设计原则：设计高内聚低耦合的系统
  4. 测试策略：理解测试的局限性与有效的测试方法选择
  5. 变更管理：如何在设计中预判和适应变化

推荐阅读

  • 《Software Engineering》by Ghezzi, Jazayeri, Mandrioli（教材）
  • 《Software Engineering》by Ian Sommerville
  • 《The Mythical Man-Month》by Fred Brooks

课外资源

  • 课程提供往届试题和练习
  • 多个Tutorial帮助理解概念
  • Assignment详细说明软件工程实践

课堂练习题精选

软件工程过程练习 (对应Topic 1)

问题： 重新排序以下开发步骤，并根据瀑布模型命名相应的阶段：

  1. 使用设计文档将整体架构划分为小模块（如用户资料模块、支付模块等）
  2. 将区块链系统投入实际使用
  3. 测试集成的区块链系统
  4. 定期接收用户反馈并改进系统
  5. 设计区块链系统的架构文档
  6. 将模块集成为集成系统
  7. 单独测试每个模块的正确性
  8. 分析和确定系统目标和需求的可行性
  9. 与用户沟通并明确区块链系统的目标和需求
  10. 开始编码：实现设计模块的结构

软件工程原则练习 (对应Topic 3)

背景： 编译器是将源代码从一种编程语言翻译成另一种语言的计算机软件。将以下开发选择(a-g)与原则匹配：

a. 将整个项目分解为可以相对容易地设计和验证的小模块
b. 使用命令行参数指定目标机器（gcc或g++编译器）
c. 正确性第一，性能其次，易用性第三
d. 编译器设计是软件开发中最严肃的课题之一
e. 新指令集配合新处理器架构
f. 为抽象机器生成中间表示/代码（如Java虚拟机）
g. 第一版只覆盖源语言的子集并省略一些功能

原则列表：

  • 严格性与形式化 (Rigor and Formality)
  • 抽象 (Abstraction)
  • 关注点分离 (Separation of Concerns)
  • 变更预判 (Anticipation of Change)
  • 模块化 (Modularity)
  • 增量性 (Incrementality)
  • 通用性 (Generality)

数据流图练习 (对应Topic 4)

使用DFD描述以下活动：

  1. 计算 A = a × b + c - d
  2. 计算 B = a × A + b × d
  3. 计算 C = A + b × B
  4. 存储并打印A、B、C的结果

有限状态机练习 (对应Topic 4)

  1. 设计一个接受字母表{0, 1}上至少包含3个1的字符串的FSM
  2. 设计一个接受包含110且以01结尾的字符串的FSM
  3. 设计一个接受1位或更多位的流，且输出满足XOR表达式(c1c2...^ci)的FSM

Petri网练习 (对应Topic 4)

使用Petri网描述哲学家就餐问题：

  • 5位哲学家交替思考和进食
  • 每人左右各有一根筷子，共5根
  • 进食需要同时拿起左右两根筷子
  • 明确标识思考/进食状态

问题： 需要多少个位置(places)？多少个转换(transitions)？初始标记(initial marking)中有多少个令牌(tokens)？

ER图练习 (对应Topic 4)

设计一个学生社团活动管理系统：

  • 记录学生ID、姓名、邮箱、专业
  • 多个活动室供学生预约
  • 记录活动室容纳人数、类别、地点、可用时间
  • 员工管理活动室使用
  • 员工有经理负责所有活动室和员工
  • 学生预约时需要提供紧急联系人（姓名和电话）

白盒测试练习 (对应Topic 6)

考试真题要点：

  1. 绘制程序的控制流图
  2. 分析控制流图
  3. 列出线性独立路径
  4. 使用正则表达式列出所有路径
  5. 描述程序功能

黑盒测试练习 (对应Topic 6)

根据系统规格说明设计测试用例，覆盖各种功能：

  • 等价类划分
  • 边界值分析
  • 决策表测试

期末考试形式

考试时间： 3小时
考试形式： 开卷（可以带教材、笔记、课件）

题型分布：

  • Question 1: 20分 - 填空题（涵盖所有主题）
  • Question 2-9: 各10分 - 问答题

题目类型：

  • 数据流图 (Data Flow Diagram)
  • Petri网 (Petri Net)
  • 逻辑规格 (Logic Specification)
  • 程序设计技术
  • UML类图和序列图
  • 白盒测试（控制流图、路径分析）
  • 黑盒测试（测试用例设计）
