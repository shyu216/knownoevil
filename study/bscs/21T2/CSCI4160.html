CSCI4160 Distributed and Parallel Computing 分布式与并行计算

课程简介 (What is this course about?)

CSCI4160 - 分布式与并行计算 是香港中文大学计算机科学与工程系的分布式与并行计算课程，涵盖现代高性能计算和分布式系统的核心概念。

课程定位：

  • 面向计算机科学与工程专业学生
  • 高性能计算的核心课程
  • 为云计算、大数据处理打下基础

知识在计算机大厦中的位置：

┌─────────────────────────────────────┐
│     云计算、大数据处理               │
├─────────────────────────────────────┤
│  ← CSCI4160: 分布式与并行计算 →      │
├─────────────────────────────────────┤

└─────────────────────────────────────┘

核心内容：

  • 并行计算：OpenMP、SIMD、GPU
  • 分布式系统：一致性、容错、消息传递
  • 性能优化：负载均衡、扩展性

未来发展方向：

  • 云计算：AWS、GCP、Azure
  • 大数据：Spark、Flink
  • 机器学习：分布式训练

📚 课程笔记 (Course Notes)

Part 1: Parallel Computing (并行计算)

Week 1-2: Parallel Background & OpenMP/SIMD

核心概念：

  • 并行计算基础：为什么需要并行？
  • 并行模式：
    • 计算并行 (Compute Parallel)
    • 数据并行 (Data Parallel)
  • 硬件发展：从多核到众核

OpenMP：

  • 共享内存并行编程接口
  • 使用 #pragma omp 指令
  • 主要概念：
    • Parallel Region
    • Work Sharing (for, sections, single)
    • Data Scope (shared, private, firstprivate)
    • Synchronization (barrier, critical, atomic)

SIMD (Single Instruction Multiple Data)：

  • 向量化执行
  • 数据级并行
  • 编译器自动向量化

难点： 理解并行与并发的区别；掌握OpenMP指令的正确使用；避免数据竞争和死锁。

Week 3: CoDesign (协同设计)

核心概念：

  • 协同设计：硬件与软件的共同优化
  • 性能优化策略：
    • 内存访问模式优化
    • 缓存利用率优化
    • 指令级并行

难点： 理解硬件架构对性能的影响；设计高效的并行算法。

Week 4: GPGPU and CUDA

CUDA编程模型：

  • 主机(Host) vs 设备(Device)：CPU与GPU的分工
  • 线程层次：
    • Grid → Block → Thread
    • Warp (32 threads)
  • 内存层次：
    • Global Memory
    • Shared Memory
    • Register
    • Constant/Texture Memory

CUDA核心概念：

  • Kernel函数 (__global__)
  • 线程同步 (__syncthreads())
  • 内存访问优化
  • 带宽优化

难点： 理解GPU架构；设计适合GPU的算法；内存访问模式优化。

Week 5: Performance Analysis & Roofline Model

性能分析方法：

  • Roofline模型：

    • 峰值计算性能 (Peak FLOPS)
    • 峰值内存带宽 (Peak Memory Bandwidth)
    • 计算 Roofline 和内存 Roofline
    • 确定性能上界

  • 性能分析工具：

    • Profilers
    • Amdahl定律
    • Gustafson定律

难点： 理解性能瓶颈；使用Roofline模型指导优化。

Week 6: MPI (Message Passing Interface)

分布式内存并行：

  • 进程间通信：
    • Point-to-point通信
    • Collective通信 (broadcast, scatter, gather, reduce)
  • 通信模式：
    • Synchronous
    • Asynchronous
  • 进程拓扑：
    • Cartesian topology
    • Graph topology

难点： 理解分布式内存模型；设计高效的通信模式；避免通信开销成为瓶颈。

Part 2: Distributed Computing (分布式计算)

Week 7: Time and Mutex (时间和互斥)

时钟与同步：

  • 物理时钟：
    • UTC, NTP
    • Clock skew vs Clock drift
  • 逻辑时钟：
    • Lamport Timestamps
    • Vector Clocks

分布式互斥：

  • 中央协调者方法：简单但有单点故障
  • 基于令牌的方法：Token Ring
  • 分布式算法：
    • Ricart-Agrawala算法
    • Maekawa投票算法

难点： 理解分布式系统中的时间概念；设计高效的分布式互斥算法。

Week 8: Leader Election and Fault Tolerance (领导选举与容错)

领导选举算法：

  • Bully算法
  • Ring算法
  • Raft中的Leader Election

容错概念：

  • 故障类型：
    • 崩溃故障 (Crash Failure)
    • 拜占庭故障 (Byzantine Failure)
  • 故障检测：
    • 心跳机制
    • Lease机制

难点： 理解不同故障模型；设计容错系统。

Week 9: Consensus and Raft (共识算法与Raft)

共识问题：

  • CAP定理：
    • Consistency (一致性)
    • Availability (可用性)
    • Partition Tolerance (分区容错性)
  • 不可能三角

Raft算法：

  • 三种角色：Leader, Candidate, Follower
  • Leader选举：
    • Term和选举超时
    • 日志复制
  • 日志复制：
    • Log Entry
    • Commit
    • Apply
  • 安全性：
    • Election Safety
    • Log Matching
    • Leader Completeness
    • State Machine Safety

Raft实现：

  • 使用gRPC进行RPC通信

难点： 理解Raft算法的每个细节；处理各种边界情况；实现高效的日志复制。

Week 10: CAP and P2P (CAP定理与点对点系统)

CAP定理深入：

  • 一致性 vs 可用性权衡
  • BASE理论：
    • Basically Available
    • Soft state
    • Eventual consistency

P2P系统：

  • 结构化P2P：DHT (Distributed Hash Table)

    • CAN
  • 非结构化P2P：Gossip协议

难点： 理解CAP权衡的实际应用；设计适合不同场景的分布式系统。

Week 11: Blockchain (区块链)

核心概念：

  • 分布式账本
  • 共识机制：
    • PoW (Proof of Work)

    • DPoS
  • 智能合约

区块链架构：

分布式系统核心概念

时间与顺序 (Time and Order)

物理时钟问题：

  • Clock Skew（时钟偏移）：不同处理器时钟值的差异
  • Clock Drift（时钟漂移）：时钟相对于真实时间的变化率
  • NTP同步无法完全消除偏移

逻辑时钟：

  • Lamport Timestamps：

    • 每个事件关联一个逻辑时间戳
    • 偏序关系：→（happens-before）
    • 规则：
      1. 同一进程中，每个事件前加1
      2. 发送消息时，时间戳+1
      3. 接收时，取max(本地时间, 消息时间戳)+1

  • Vector Clocks：

    • 每个进程维护一个向量
    • 向量[i] = 进程i已发生的事件数
    • 比Lamport Timestamp更能检测并发

互斥算法

Ricart-Agrawala算法：

  • 全分布式互斥算法
  • 需要2(N-1)条消息
  • 基于Lamport逻辑时钟

Maekawa投票算法：

  • 改进的投票算法
  • 需要√N量级消息
  • 使用 quorum 机制

共识问题

CAP定理：

  • Consistency（一致性）
  • Availability（可用性）
  • Partition Tolerance（分区容错性）
  • 只能同时满足两个！

Raft算法实现细节：

Leader Election：

  1. 选举超时（Election Timeout）：150-300ms随机
  2. 转换为Candidate，增加Term
  3. 给自己投票，发送RequestVote
  4. 获得多数票则成为Leader

Log Replication：

  1. Leader接收客户端请求
  2. 添加到本地日志
  3. 发送AppendEntries给Followers
  4. 多数确认后commit
  5. 应用到状态机

安全性保证：

  • Election Safety：同一Term最多一个Leader
  • Leader Append-Only：Leader从不覆盖或删除日志
  • Log Matching：如果两个日志有相同索引和Term，则日志相同
  • Leader Completeness：当选Leader包含所有committed日志
  • State Machine Safety：应用相同顺序的日志产生相同状态

并行计算核心概念

OpenMP详解

并行区域 (Parallel Region)：

#pragma omp parallel
{
    // 并行执行
}

工作共享 (Work Sharing）：

#pragma omp for
for (int i = 0; i < N; i++) {
    // 并行处理
}

#pragma omp sections
{
    #pragma omp section
    // 任务1
    
    #pragma omp section
    // 任务2
}

数据作用域 (Data Scope)：

  • shared：所有线程共享
  • private：每个线程独立副本
  • firstprivate：私有但初始化为原值
  • reduction：归约操作

同步：

  • barrier：所有线程等待
  • critical：互斥访问
  • atomic：原子操作

CUDA详解

内存层次：

  • Global Memory：最大但最慢
  • Shared Memory：每个block共享，快速
  • Register：最快，每个线程私有
  • Constant/Texture：只读，缓存

优化技巧：

  • 合并内存访问 (Coalesced Memory Access)
  • 避免bank conflict
  • 使用shared memory
  • 合理设置block大小

MPI详解

点对点通信：

  • MPI_Send：同步发送
  • MPI_Isend：异步发送
  • MPI_Recv：接收
  • MPI_Sendrecv：双向通信

集合通信：

  • MPI_Broadcast：广播
  • MPI_Scatter：分发
  • MPI_Gather：收集
  • MPI_Reduce：归约
  • MPI_Allreduce：全归约

  4. OpenMP指令：各种pragma的使用
  5. CUDA内存模型：全局内存、共享内存优化
  6. MPI通信模式：点对点和集体通信
