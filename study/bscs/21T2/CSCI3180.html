CSCI3180 Principles of Programming Languages 编程语言原理

Dale2025/5/11大约 10 分钟CUHKCourse21T2Computer ScienceProgramming LanguagesTheory

课程简介 (What is this course about?)

CSCI3180 - 编程语言原理 是香港中文大学计算机科学与工程系的编程语言原理课程，旨在帮助学生理解编程语言的设计原理、实现机制和评估标准。

课程定位：

  • 面向计算机科学与工程专业学生
  • 编程语言理论的核心课程
  • 为编译器设计、语言设计打下基础

知识在计算机大厦中的位置：

┌─────────────────────────────────────┐
│     编译器设计、语言设计             │
├─────────────────────────────────────┤
│  ← CSCI3180: 编程语言原理 →          │
├─────────────────────────────────────┤
│     形式语言与自动机、数据结构        │
└─────────────────────────────────────┘

核心内容：

  • 语言设计：语法、语义、类型系统
  • 编程范式：函数式、面向对象、逻辑编程
  • 实现技术：解释器、编译器

未来发展方向：

  • 编译器设计：LLVM、GCC
  • 语言设计：DSL、新语言开发
  • 类型理论：依赖类型、线性类型

📚 课程笔记 (Course Notes)

Chapter 1: Preliminaries (导论)

核心内容：

  • 学习编程语言的理由：
    • 提高表达想法的能力
    • 更好地选择适合的语言
    • 更容易学习新语言
    • 深入理解实现细节
    • 促进计算领域的整体进步

编程领域 (Programming Domains)：

  • 科学应用：大量浮点计算 → Fortran
  • 商业应用：报告生成、十进制数、字符 → COBOL
  • 人工智能：符号处理 → LISP, Prolog
  • 系统编程：效率优先 → C
  • Web软件：混合语言（XHTML, PHP, Java）

语言评估标准：

  1. 可读性 (Readability)：程序易于阅读和理解的难易程度
  2. 可写性 (Writability)：使用语言创建程序的难易程度
  3. 可靠性 (Reliability)：符合规格说明的程度
  4. 成本 (Cost)：总体拥有成本

可读性详细标准：

  • 整体简单性：可管理的特性集、最小化操作符重载
  • 正交性：少量原语可组合的方式
  • 控制语句：清晰的控制结构
  • 数据类型：充分的数据结构定义设施
  • 语法考虑：标识符形式、自描述构造

可写性详细标准：

  • 简单性和正交性
  • 抽象支持：定义和使用复杂结构的能力
  • 表达性：便捷的运算指定方式

可靠性详细标准：

  • 类型检查：类型错误检测
  • 异常处理：运行时错误拦截
  • 别名：同一内存位置的多重引用

编程语言设计影响

计算机架构影响：

  • 冯·诺依曼架构：命令式语言的基础
    • 数据和程序存储在内存中
    • 内存与CPU分离
    • 指令和数据从内存传输到CPU
    • 变量模拟存储单元
    • 赋值语句模拟数据传输
    • 迭代效率高

冯·诺依曼瓶颈：

  • 内存与处理器之间的连接速度决定计算机速度
  • 程序指令执行速度常快于连接速度，形成瓶颈

编程方法论影响：

  • 1950s-1960s初：简单应用，关注机器效率
  • 1960s末：人的效率重要化 → 结构化编程
  • 1970s末：面向数据 → 数据抽象
  • 1980s中：面向对象编程 → 数据抽象+继承+多态

编程语言分类

  1. 命令式 (Imperative)

    • 核心特性：变量、赋值语句、迭代
    • 示例：C, Pascal

  2. 函数式 (Functional)

    • 主要计算方式：函数应用
    • 示例：LISP, Scheme, ML

  3. 逻辑式 (Logic)

    • 基于规则（无特定顺序）
    • 示例：Prolog

  4. 面向对象 (Object-Oriented)

    • 数据抽象、继承、迟绑定
    • 示例：Java, C++

  5. 标记语言 (Markup)

    • 非编程语言，用于Web文档布局
    • 示例：XHTML, XML

抽象 (Abstraction)

抽象的定义：

  • 抽象是一种思维方式，专注于一般概念而非具体表现形式
  • 原因：人脑能处理的复杂度远低于编写简单软件所需

抽象示例：

  • 变量和赋值抽象了存储获取和存储
  • 控制结构抽象了跳转
  • 泛型抽象了程序中对值类型的操作

抽象类型：

  • 数据抽象：一组对象及其操作特征
  • 控制抽象：任意操作序列化的方法
  • 过程和模块抽象：计算动作的规范

实现方法

编译 (Compilation)：

  • 将高级语言程序翻译成机器语言
  • 翻译慢，执行快
  • 编译过程阶段：
    • 词法分析：将字符转换为词法单元
    • 语法分析：转换为语法树
    • 语义分析：生成中间代码
    • 代码生成：生成机器码

纯解释 (Pure Interpretation)：

  • 无翻译过程
  • 易于调试
  • 执行慢（10-100倍于编译程序）
  • JavaScript等Web脚本语言使用

混合实现系统 (Hybrid Implementation)：

  • 编译和解释的折中
  • 翻译为中间语言以便解释
  • 示例：Java字节码（Java Virtual Machine）

即时编译 (JIT - Just-in-Time)：

  • 初始翻译为中间语言
  • 然后编译为机器码
  • 保留机器码版本供后续调用
  • Java和.NET语言广泛使用

语言设计权衡

  • 可靠性 vs 执行成本：Java数组索引检查
  • 可读性 vs 可写性：APL强大但难读
  • 可写性 vs 可靠性：C++指针灵活但不可靠

数据抽象 (Data Abstraction)

主要内容：

  • 值 (Values)
  • 表达式 (Expressions)
  • 类型 (Types)
  • 类型系统 (Type Systems)
  • 变量 (Variables)

重点概念：

  • 静态类型 vs 动态类型
  • 强类型 vs 弱类型
  • 类型检查的时机（编译时 vs 运行时）

控制抽象 (Control Abstraction)

主要内容：

  • 选择 (Selection)
  • 迭代 (Iteration)
  • 递归 (Recursion)
  • 异常 (Exception)
  • 并发控制 (Concurrency Control)

模块抽象 (Modular Abstraction)

主要内容：

  • 过程和函数 (Procedures and Functions)
  • 参数传递 (Parameter Passing)
  • 副作用 (Side Effects)
  • 块 (Blocks)
  • 绑定和作用域 (Bindings and Scoping)

声明式编程范式 (Declarative Programming Paradigm)

逻辑编程：

  • 如何用逻辑表达
  • Prolog作为代表

函数式编程：

  • ML作为可编程计算器
  • 高阶函数
  • 模式匹配

评估方式

  • 作业 (60-70%)：

    • 多个编程作业
    • 实现不同范式的语言特性

  • 期末考试 (30-40%)：涵盖所有主题

课程难点总结

  1. 类型系统理解：静态类型与动态类型的区别、类型检查机制
  2. 作用域规则：静态作用域 vs 动态作用域、闭包
  3. 参数传递机制：值传递、引用传递、名字调用、延迟求值
  4. 多态性：参数多态、特设多态、子类型多态、泛型
  5. 抽象机制：数据抽象、过程抽象、控制抽象的理解与应用
  6. 不同范式的思维方式：从命令式到函数式、逻辑式的转变

  • 《Programming Language Pragmatics》by Michael Scott
  • 《Design Concepts in Programming Languages》by Reynolds

实践内容

  • 使用不同编程范式完成编程作业
  • 分析语言特性并理解其设计动机
  • 比较不同语言的设计选择

深入主题：名称、绑定与作用域 (Chapter 5)

名称 (Names)

名称设计问题：

  • 是否区分大小写？
  • 保留字 vs 关键字

名称长度限制：

  • FORTRAN 95：最多31个字符
  • C99：无限制，但前63个字符有效；外部名称最多31个
  • C#, Ada, Java：无限制，所有字符都有效
  • C++：无限制，但实现者通常会施加限制

特殊字符规则：

  • PHP：所有变量名必须以$开头
  • Perl：所有变量名以特殊字符开头，指定变量类型
  • Ruby：以@开头的是实例变量；@@开头的是类变量

保留字 vs 关键字：

  • 关键字：在特定上下文中才是特殊的词
  • 保留字：不能用作用户定义的名称
  • COBOL有300个保留字！

变量 (Variables)

变量是内存单元的抽象，属性包括：

  1. 名称 (Name) - 不是所有变量都有
  2. 地址 (Address) - 变量关联的内存地址
    • 同一变量在不同时间可能有不同地址
    • 两个变量名可以访问同一内存位置，称为别名 (Alias)
    • 别名通过指针、引用变量、C/C++联合创建
    • 别名损害可读性
  3. 值 (Value) - 变量关联位置的内容
    • l-value：地址
    • r-value：值
  4. 类型 (Type) - 决定值范围和操作集合
  5. 生命周期 (Lifetime) - 绑定到特定内存单元的时间
  6. 作用域 (Scope) - 程序中可见的区域

绑定 (Binding)

绑定时间：

  • 语言设计时：操作符符号绑定到操作
  • 语言实现时：浮点类型绑定到表示
  • 编译时：变量绑定到类型（C, Java）
  • 加载时：静态变量绑定到内存单元
  • 运行时：非静态局部变量绑定到内存单元

静态绑定 vs 动态绑定：

  • 静态绑定：发生在运行前且保持不变
  • 动态绑定：发生在执行期间或可在执行期间改变

类型绑定 (Type Binding)

显式声明 vs 隐式声明：

  • 显式声明：使用声明语句
  • 隐式声明：默认机制（Fortran, BASIC, Perl, Ruby, JavaScript, PHP）
    • 优点：可写性
    • 缺点：可靠性降低

类型推断 (Type Inference)：

  • C#：var关键字
  • ML, Haskell, F#, Go：根据上下文推断类型

动态类型绑定：

  • JavaScript, Python, Ruby, PHP
  • 通过赋值语句指定类型
  • 优点：灵活性
  • 缺点：高成本、编译器难以检测类型错误

变量按生命周期分类

  1. 静态 (Static)：执行前绑定到内存单元，整个执行期间保持绑定
  2. 栈动态 (Stack-Dynamic)：在声明时绑定，需要显式分配和释放
  3. 堆动态 (Heap-Dynamic)：通过显式指令分配和释放

深入主题：数据类型 (Chapter 6)

原始数据类型

整数类型 (Integer)：

  • 几乎总是硬件的直接反映
  • Java有8种整数类型：byte, short, int, long

浮点类型 (Floating Point)：

  • 近似实数
  • IEEE 754标准

复数类型 (Complex)：

  • C99, Fortran, Python支持
  • 每个值由两个浮点数组成（实部和虚部）

十进制类型 (Decimal)：

  • 商业应用（货币）
  • COBOL必需
  • 以BCD形式存储，精确但范围有限

布尔类型 (Boolean)：

  • 两个值：true和false
  • 通常用字节实现以提高可读性

字符类型 (Character)：

  • ASCII编码（最常用）
  • Unicode（16位）
  • UTF-32（32位）

字符串类型 (Character String)

设计问题：

  • 是原始类型还是特殊数组？
  • 长度静态还是动态？

长度选项：

  1. 静态：COBOL, Java String类
  2. 有限动态：C/C++，使用特殊字符表示字符串结束
  3. 动态：SNOBOL4, Perl, JavaScript

典型操作：

  • 赋值和复制
  • 比较
  • 连接
  • 子串引用
  • 模式匹配

用户定义有序类型

枚举类型 (Enumeration)：

  • 所有可能值在定义中列出
  • C#示例：enum days {Mon, Tue, Wed, Thu, Fri, Sat, Sun}

设计问题：

  • 枚举常量能否出现在多个类型定义中？
  • 枚举值是否隐式转换为整数？

数组类型 (Array)

设计问题：

  • 索引类型和范围？
  • 何时进行范围检查？
  • 数组作为参数时是否传递长度？

关联数组 (Associative Arrays)

  • 键-值对集合
  • Perl, Python, Ruby, JavaScript支持
  • 允许任意类型作为索引

记录类型 (Record)

  • 异构数据的聚合
  • 字段访问
  • COBOL和Pascal典型支持

联合类型 (Union)

  • 同一内存位置可存储不同类型
  • C/C++：union
  • Ada：variant record

指针和引用类型 (Pointer and Reference)

设计问题：

  • 指针类型是否受约束？
  • 指针是否可算术运算？
  • 垃圾回收vs显式释放？

引用类型：

  • Java, C#：引用类型
  • 不能直接操作内存地址

类型检查 (Type Checking)

  • 强类型：所有类型错误都能被检测
  • 弱类型：允许某些类型违规
  • 静态类型检查 vs 动态类型检查

类型等价 (Type Equivalence)

  • 结构等价：类型具有相同结构
  • 名字等价：类型具有相同名字

考试形式与作业

评估方式：

  • 作业 (60-70%)：多个编程作业
  • 期末考试 (30-40%)

考试重点：

  • 理解不同编程语言的设计权衡
  • 分析语言特性
  • 解释语言特性对程序行为的影响
