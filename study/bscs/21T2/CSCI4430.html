📚 课程笔记 (Course Notes)

什么是计算机网络？

计算机网络是由"链路"(links)互联"节点"(nodes)的系统，目的是在节点之间传输"信息"。本课程主要关注互联网：

  • 两台机器 → 链路 (link)
  • 多台机器 → 网络 (network)
  • 多个网络 → 互联网 (Internet)

本课程不涉及：

  • 如何设计网页
  • 如何编写Web服务

课程大纲

Lecture 1: Introduction and Overview (导论)

核心概念：

  • 网络发展历史：
    • 1969: ARPAnet，第一条链路（UCLA ↔ SRI）
    • 1971: 多节点，MAC层问题
    • 1980: 路由问题，网络层
    • 1980s: 带宽共享，传输层
    • 现在: 应用层服务

网络评估指标：

  • 延迟 (Delay)：数据包从源到目的地的时间

    • 传输延迟 (Transmission Delay)
    • 传播延迟 (Propagation Delay)
    • 排队延迟 (Queuing Delay)
    • 处理延迟 (Processing Delay)

  • 丢包 (Loss)：数据包被丢弃的比例

  • 吞吐量 (Throughput)：目的地接收数据的速率

  • 带宽-延迟积 (BDP)：任何时刻"在飞行中"的比特数

共享网络的方式：

  1. 电路交换 (Circuit Switching)：

    • 预留资源 per 连接
    • 优点：可预测性能、简单快速
    • 缺点：建立/拆除复杂、效率低（突发流量）

  2. 分组交换 (Packet Switching)：

    • 按需分配资源
    • 优点：高效利用资源、简单、鲁棒
    • 缺点：不可预测、需要缓冲管理和拥塞控制

统计复用 (Statistical Multiplexing)：

  • 允许需求超过网络处理能力
  • 假设不是所有需求同时需要
  • 结果：不可预测性

难点： 理解电路交换与分组交换的权衡；掌握各种延迟的组成。

  • CDN (内容分发网络)：

    • 地理分布的服务器
    • 负载均衡
    • 缓存策略

网络应用架构：

  • 客户端-服务器 (Client-Server)
  • P2P (Peer-to-Peer)
  • 混合架构

难点： 理解DNS的层次结构和查询过程；掌握CDN的工作原理。

核心概念：

  • 视频流：

    • DASH (Dynamic Adaptive Streaming over HTTP)

    • 转码和码率适配

  • 云计算：

    • IaaS, PaaS, SaaS

难点： 理解视频流的自适应机制；掌握云计算的基本概念。

Lecture 5-6: Transport Layer (传输层)

核心概念：

  • UDP (用户数据报协议)：

    • 无连接
    • 不可靠但高效
    • 适用场景：实时应用、DNS、VoIP

  • TCP (传输控制协议)：

    • 面向连接
    • 可靠传输
    • 拥塞控制
    • 流量控制

TCP详细机制：

  • 三次握手：

    1. SYN
    2. SYN-ACK
    3. ACK

  • 四次挥手：

    1. FIN
    2. ACK
    3. FIN
    4. ACK

  • 拥塞控制：

    • 慢启动 (Slow Start)
    • 拥塞避免 (Congestion Avoidance)
    • 快速重传 (Fast Retransmit)
    • 快速恢复 (Fast Recovery)

  • 流量控制：

    • 滑动窗口
    • 接收窗口

难点： 深入理解TCP的可靠传输机制；掌握拥塞控制算法。

性能优化：

  • TCP参数调优：调整拥塞窗口大小、超时重传时间等参数
  • 路径MTU发现：避免IP分片，提高传输效率
  • 延迟ACK：减少ACK报文数量，降低网络开销
  • 快速打开（Fast Open）：减少连接建立时间
  • 批量数据传输：使用TCP_NODELAY禁用Nagle算法，优化实时应用

安全机制：

  • TCP SYN cookies：防止SYN洪水攻击
  • 连接限制：限制单个IP的并发连接数
  • 流量监控：检测异常流量模式，防止DoS攻击
  • 加密传输：使用TLS/SSL保护传输数据
  • 防火墙规则：过滤恶意TCP流量

Lecture 7-8: Congestion Control (拥塞控制)

核心概念：

  • 拥塞原因：
    • 太多数据注入网络
    • 路由器缓冲溢出
    • 资源竞争

TCP拥塞控制算法：

  • AIMD (Additive Increase Multiplicative Decrease)

  • ** Tahoe 和 Reno**：

    • 慢启动阈值 (ssthresh)
    • 三个重复ACK → 快速重传
    • 超时 → 回到慢启动

  • NewReno：

    • 部分ACK
    • 快速恢复改进

  • CUBIC：

    • 当前Linux默认
    • W(t) = C(t - K)³ + Wmax

难点： 理解各种拥塞控制算法的区别；分析网络性能。

Lecture 9-10: Network Layer (网络层)

核心概念：

  • IP协议：

    • IP地址
    • 子网掩码
    • CIDR
    • NAT

  • 路由算法：

    • 链路状态 (Link State)：OSPF
    • 距离向量 (Distance Vector)：RIP

  • 路由器：

    • 转发 (Forwarding)
    • 路由 (Routing)
    • 输入/输出排队

IPv6：

  • 地址空间扩大
  • 简化的头部
  • 内置安全

难点： 理解IP协议的工作原理；掌握路由算法。

性能优化：

  • 路由优化：选择最短路径，减少网络延迟
  • 负载均衡：在多路径之间分配流量，提高网络利用率
  • 路由聚合：减少路由表大小，提高路由查找速度
  • 快速转发：使用硬件加速，提高数据包处理速度
  • 流量工程：优化流量分布，避免网络拥塞

安全机制：

  • IPsec：提供IP层的加密和认证
  • 防火墙：过滤非法IP数据包
  • IDS/IPS：检测和防御网络入侵
  • 源地址验证：防止IP地址欺骗
  • 网络分段：将网络划分为不同安全区域

Lecture 11-12: Routing (路由协议)

核心概念：

  • 域内路由 (Intra-AS)：

    • OSPF (Link State)
    • RIP (Distance Vector)

  • 域间路由 (Inter-AS)：

    • BGP (Border Gateway Protocol)
    • AS路径向量
    • 策略路由

BGP详细机制：

  • eBGP 和 iBGP
  • 路由属性：
    • AS-PATH
    • NEXT-HOP
    • LOCAL-PREF
    • MED
  • 路由选择：
    1. 最高LOCAL-PREF
    2. 最短AS-PATH
    3. 最低MED
    4. 最近下一跳
    5. BGP ID

难点： 理解BGP的复杂性和策略性；掌握路由选择过程。

Lecture 15-17: Link Layer and LAN (链路层)

核心概念：

  • MAC协议：

    • CSMA/CD (以太网)
    • CSMA/CA (WiFi)
    • ALOHA

  • 以太网 (Ethernet)：

    • 帧结构
    • ARP协议
    • 交换机学习

  • WiFi (802.11)：

    • CSMA/CA
    • RTS/CTS
    • 隐藏终端问题

  • PPP协议：

    • 链路控制协议 (LCP)
    • 网络控制协议 (NCP)

难点： 理解MAC层协议的工作原理；掌握交换机的工作方式。

性能优化：

  • 链路聚合：将多个物理链路绑定为一个逻辑链路，提高带宽
  • VLAN：减少广播域，提高网络效率
  • 端口镜像：用于网络监控，不影响正常流量
  • QoS：为不同类型的流量分配不同的优先级
  • 快速生成树协议（RSTP）：减少网络收敛时间

安全机制：

  • MAC地址过滤：限制只有特定MAC地址的设备可以访问网络
  • 802.1X：基于端口的访问控制
  • 端口安全：限制每个端口的MAC地址数量
  • 广播风暴抑制：防止广播风暴导致网络拥塞
  • ARP欺骗防护：防止ARP缓存中毒攻击

🚀 实际应用案例

案例1：企业网络性能优化

  • 背景：某企业内部网络延迟高，影响业务系统运行
  • 问题分析：网络拓扑不合理，带宽分配不均
  • 解决方案：
    • 优化网络拓扑，采用三层架构
    • 实施QoS，为关键业务流量分配更高优先级
    • 部署负载均衡器，分散流量压力
  • 效果：网络延迟减少60%，业务系统响应速度提升40%

案例2：数据中心网络设计

  • 背景：大型互联网公司需要构建高性能数据中心网络
  • 问题分析：传统三层架构无法满足大规模流量需求
  • 解决方案：
    • 采用Spine-Leaf架构，提高网络吞吐量
    • 部署SDN，实现网络可编程性
    • 使用BGP EVPN，简化网络管理
  • 效果：网络带宽提升3倍，故障恢复时间从分钟级降至秒级

案例3：网络安全防护

  • 背景：企业网络遭受DDoS攻击，服务中断
  • 问题分析：缺乏有效的DDoS防护机制
  • 解决方案：
    • 部署DDoS清洗服务，过滤恶意流量
    • 实施源地址验证，防止IP欺骗
    • 配置防火墙规则，限制异常流量
  • 效果：成功抵御多次DDoS攻击，服务可用性保持99.99%

补充：网络分层与协议

为什么要分层？

分层思想：
想象CEO A给CEO B写信的邮递过程：

  1. CEO层：写语义内容（信件内容）
  2. 助理层：装信封，识别收件人身份
  3. 快递公司层：定位地址，打包运输

分层的优势：

  1. 分解问题：将复杂问题分解为可管理的任务
  2. 组织任务：将任务组织成层次结构
  3. 职责分配：每个实体明确自己的职责

关键概念：

  • 对等实体 (Peer Entities)：同一层的实体可以相互理解
  • 封装 (Encapsulation)：每层添加自己的头部信息
  • 服务访问点 (SAP - Service Access Point)：层与层之间的接口

OSI七层模型 vs TCP/IP四层模型

OSI七层：

  1. 物理层 (Physical)
  2. 数据链路层 (Data Link)
  3. 网络层 (Network)
  4. 传输层 (Transport)
  5. 会话层 (Session)
  6. 表示层 (Presentation)
  7. 应用层 (Application)

TCP/IP四层：

  1. 网络接口层 (Link)
  2. 网络层 (Internet)
  3. 传输层 (Transport)
  4. 应用层 (Application)

协议 (Protocol)

协议定义：

  • 控制两个对等实体通信的规则集合
  • 包括语法（格式）、语义（含义）、时序（顺序）

协议分层示例：

  • 应用层：HTTP, SMTP, FTP, DNS
  • 传输层：TCP, UDP
  • 网络层：IP, ICMP
  • 数据链路层：Ethernet, WiFi, PPP

补充：传输层详解

UDP vs TCP详细对比

UDP特性：

  • 无连接：不需要握手
  • 不可靠：不保证交付、不按序、不去重
  • 简单高效：头部仅8字节
  • 无拥塞控制
  • 支持一对一、一对多、多对一、多对多

UDP使用场景：

  • DNS查询
  • VoIP/IP电话
  • 视频流（容忍少量丢包）
  • 实时游戏

TCP详细机制

三次握手详解：

客户端                    服务端
  |                        |
  |-------- SYN ----------->|  客户端发送SYN，设置seq=x
  |<------ SYN-ACK ---------|  服务端发送SYN+ACK，seq=y, ack=x+1
  |-------- ACK ----------->|  客户端确认，seq=x+1, ack=y+1
  |                        |
  |------ [数据传输] ------>|  连接建立，开始传输数据

四次挥手详解：

客户端                    服务端
  |                        |
  |-------- FIN ---------->|  客户端发送FIN，进入FIN-WAIT-1
  |<------ ACK ------------|  服务端确认，进入CLOSE-WAIT
  |                        |  客户端进入FIN-WAIT-2
  |<------ FIN ------------|  服务端准备好关闭，发送FIN
  |-------- ACK ----------->|  客户端确认，进入TIME-WAIT
  |                        |  等待2MSL后关闭

滑动窗口机制：

  • 发送方维持一个发送窗口
  • 窗口内的数据可以发送
  • 收到确认后窗口滑动
  • 接收方维持一个接收窗口

拥塞控制算法详解：

慢启动 (Slow Start)：

  • 初始拥塞窗口(cwnd)=1 MSS
  • 每收到一个ACK，cwnd翻倍
  • 指数增长，直到达到ssthresh

拥塞避免 (Congestion Avoidance)：

  • cwnd达到ssthresh后
  • 每收到一个ACK，cwnd增加1 MSS
  • 线性增长

快速重传 (Fast Retransmit)：

  • 收到3个重复ACK
  • 立即重传丢失的段
  • 不等待超时

快速恢复 (Fast Recovery)：

  • 快速重传后
  • ssthresh = cwnd / 2
  • cwnd = ssthresh + 3
  • 进入拥塞避免

TCP Reno vs NewReno vs CUBIC

TCP Reno：

  • 快速重传+快速恢复
  • 问题：多个丢包可能触发多次慢启动

NewReno：

  • 改进快速恢复
  • 跟踪每个SACK块
  • 部分ACK不退出恢复

CUBIC (Linux默认)：

  • 使用三次函数计算窗口
  • W(t) = C(t - K)³ + Wmax
  • 快速恢复带宽

IP协议

IPv4头部字段：

  • 版本(4位)：IPv4=4
  • 头部长度(4位)：以4字节为单位
  • 服务类型(8位)：QoS
  • 总长度(16位)：IP包总长度
  • 标识(16位)：分片重组用
  • 标志(3位)：DF, MF
  • 片偏移(13位)
  • TTL(8位)：跳数限制
  • 协议(8位)：TCP=6, UDP=17, ICMP=1
  • 头部校验和(16位)
  • 源IP地址(32位)
  • 目的IP地址(32位)

IPv6改进：

  • 地址空间：128位
  • 简化的头部（40字节固定）
  • 内置安全(IPsec)
  • 自动配置
  • 支持移动性

路由算法

链路状态算法 (OSPF)：

  1. 发现邻居
  2. 测量到邻居的代价
  3. 构造LSA
  4. 扩散LSA
  5. 计算Dijkstra最短路

距离向量算法 (RIP)：

  • 使用Bellman-Ford
  • 每30秒广播整个路由表
  • 最大跳数15

BGP

BGP消息类型：

  • OPEN：建立连接
  • UPDATE：更新路由
  • NOTIFICATION：报错
  • KEEPALIVE：保活

BGP路由属性：

  • AS-PATH：经过的AS序列
  • NEXT-HOP：下一跳IP
  • LOCAL-PREF：本地优先级
  • MED：多出口鉴别器

补充：链路层与局域网

MAC协议

CSMA/CD (以太网)：

  • 载波侦听 (Carrier Sense)
  • 冲突检测 (Collision Detection)
  • 指数退避 (Exponential Backoff)

CSMA/CA (WiFi)：

  • 载波侦听
  • 冲突避免
  • RTS/CTS机制

ARP协议

ARP功能：

  • IP地址到MAC地址的映射
  • 广播查询，单播响应

ARP表：

  • 缓存IP-MAC映射
  • 典型TTL：20分钟

交换机

交换机学习：

  • 记录每个MAC地址对应的端口
  • 转发基于MAC地址
  • 泛洪未知目的地址

补充：应用层协议

HTTP

HTTP请求方法：

  • GET：请求资源
  • POST：提交数据
  • PUT：上传资源
  • DELETE：删除资源

HTTP状态码：

  • 1xx：信息
  • 2xx：成功
  • 3xx：重定向
  • 4xx：客户端错误
  • 5xx：服务器错误

Cookie机制：

  • 服务器生成Cookie
  • 存储在客户端
  • 每次请求携带
  • 用于会话管理

DNS

DNS查询流程：

客户端 → 本地DNS服务器 → 根DNS服务器 → TLD DNS服务器 → 权威DNS服务器

DNS记录类型：

  • A：域名到IPv4
  • AAAA：域名到IPv6
  • CNAME：别名
  • MX：邮件服务器
  • NS：域名服务器
  • SOA：授权起始

CDN

CDN工作原理：

  1. 用户请求资源
  2. DNS解析到最近的CDN节点
  3. CDN节点检查缓存
  4. 缓存命中则返回
  5. 否则从源站拉取并缓存

负载均衡策略：

  • 地理最近
  • 负载最低
  • 延迟最小
