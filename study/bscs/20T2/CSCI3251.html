课程简介 (What is this course about?)

CSCI3251 - 工程实践 是香港中文大学计算机科学与工程系的实践课程，与 CSCI3250 配合开设。这门课程旨在让学生获得计算机和工程的实际技能实践经验。

课程定位：

  • 面向计算机科学与工程专业学生
  • 培养实际工程技能
  • 理解团队协作与版本控制

核心内容：

  • Linux 系统：命令行操作、Shell 脚本
  • 版本控制：Git、GitHub 工作流
  • 开发工具：编辑器、IDE、正则表达式
  • 深度学习：Keras 入门

未来发展方向：

  • DevOps：自动化部署、CI/CD
  • 云计算：Docker、Kubernetes
  • 机器学习工程：模型部署与维护

Workshop 1: Ubuntu虚拟机安装与CSE Linux服务器连接

本Workshop内容包括：

Workshop 2: 命令工具与脚本编程

本Workshop系统介绍Shell和脚本编程：

  1. Shell基础

    • Shell是从键盘获取命令并交给操作系统执行的程序
    • 常用Shell：bash, zsh, ksh, tcsh
    • 终端(Terminal)是打开窗口与Shell交互的程序

  2. 基本命令

    • 文件/目录命令：pwd, ls, cd, mkdir, cp, rm, mv
    • 搜索命令：find, grep
    • 进程管理：top, kill
    • 系统命令：help, history, date

  3. 权限管理

    • Linux权限分为三類：User(所有者)、Group(组)、Others(其他人)
    • 权限标志：r(读)、w(写)、x(执行)
    • 使用chmod命令修改权限
    • 示例：chmod 754 test.log 表示所有者有读写执行权限，组有读执行权限，其他人有读权限

  4. 基础脚本编程

    • 变量声明：NAME=value（等号两边不能有空格）
    • 变量引用：使用$符号，如echo $NAME
    • 循环结构：while循环和for循环
    • shebang：#!/bin/bash必须放在脚本第一行

难点解析：

  • 理解权限的八进制表示法
  • 掌握shell脚本的基本语法

Workshop 3: 版本控制与Git基础

本Workshop由LAI Xin主讲，系统介绍版本控制系统和Git：

1. 版本控制系统 (Version Control Systems)

  • 什么是版本控制？

    • 允许多人同时工作于源代码
    • 每人维护自己的副本，不干扰他人
    • 允许一人使用多台电脑
    • 整合不同团队成员的工作
    • 可访问历史版本

  • 版本控制系统的类型

    • 集中式 (Centralized)：只有一个仓库，如Subversion
    • 分布式 (Distributed)：多个仓库，如Git、Mercurial
    • 分布式VCS的优势：可离线工作、稳定的多重备份、更快的命令执行

2. Git基础

  • Git与Github的关系

    • Git是免费开源的分布式版本控制系统
    • Github提供基于Git的项目托管服务

  • Git的三个"树"

    • HEAD：本地仓库的当前版本
    • Index（Staging area）：暂存区域，预备下次提交
    • Working directory：个人工作副本

  • 基本工作流程

    1. 编辑文件（在Working Directory）
    2. 暂存更改（从Working Directory到Staging area）
    3. 提交更改（从Staging area到Local Repository）

  • 基本Git命令

    • git init：初始化仓库
    • git clone <url>：克隆远程仓库
    • git add <filename>：添加文件到暂存区
    • git commit -m "xxx"：提交更改
    • git push：推送到远程仓库
    • git pull：拉取远程仓库并合并
    • git status：查看状态
    • git diff：查看更改
    • git log：查看提交历史
    • git checkout <commit>：回滚到指定版本
    • git branch：创建/查看分支
    • git merge：合并分支

难点解析：

  • 理解仓库(Repository)和工作副本(Working Copy)的关系
  • 掌握分支(Branch)的创建和合并
  • 学会解决合并冲突

Workshop 4: Github与Github Actions

本Workshop由ZHENG Wu主讲：

1. Github基本功能

  • 创建仓库 (Create Repository)

  • 分支管理 (Branching)

    • 从master/main分支创建新分支进行开发
    • 完成后合并回主分支

  • 提交更改与Pull Request

    • 在特性分支上修改和提交
    • 发起Pull Request与团队成员讨论
    • 审核通过后合并到主分支

  • Issues问题跟踪

    • 提交问题或疑问
    • 项目创建者进行解答
    • 良好的问题处理可以关闭Issue

  • 项目管理功能

    • 使用Kanban板跟踪任务进度
    • 计划和管理项目任务

2. Github Actions自动化

  • 持续集成/持续部署(CI/CD)
  • 自动测试和部署工作流

难点解析：

  • 理解Pull Request的工作流程
  • 掌握分支合并的时机和方法

Workshop 5: 编辑器、IDE与正则表达式

本Workshop由Xinyan DAI主讲：

1. 文本编辑器

  • 常见编辑器：vi/vim, gEdit, VSCode, GitHub Atom
  • 文本编辑器的作用：
    • 语法高亮
    • 语法检查
    • 代码补全
    • 代码搜索和重命名

2. Vim编辑器

  • 为什么学习Vim？

    • 任何UNIX系统默认安装
    • 在数据中心通常只有文本控制台
    • 轻量、快速、高效

  • Vim的三种模式

    • 普通模式 (Normal Mode)：导航和操作文本，按Esc进入
    • 插入模式 (Insert Mode)：输入文本，按i进入
    • 可视模式 (Visual Mode)：选择文本，按v进入

  • 基本命令

    • :w - 保存
    • :wq - 保存并退出
    • :q! - 不保存退出
    • 移动：h(左)、j(下)、k(上)、l(右)

3. 正则表达式 (Regex)

  • 模式匹配和文本搜索的强大工具
  • 用于数据验证、文本处理等场景

难点解析：

  • 熟悉Vim的操作模式切换
  • 掌握正则表达式的基本语法

Q&A

Q: 这门课需要什么先修课程吗？
A: 不需要特定的先修课程，但建议有一定的计算机基础。

Q: Workshop提交有什么要求？
A: 每个Workshop需要按要求提交相关文件和截图到Blackboard，截止日期通常为周五23:59。

Q: 如何连接CSE Linux服务器？
A: 在校内可直接SSH连接；在校外需要先设置VPN或使用SSH Gateway (gw.cse.cuhk.edu.hk)。

Q: 推荐什么工具进行SSH连接？
A: Windows用户推荐PuTTY或Windows Terminal；macOS和Linux用户可直接使用终端。
