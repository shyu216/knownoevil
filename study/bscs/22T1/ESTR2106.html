📚 课程大纲与每节课内容

Week 1: Introduction, HTML, Bootstrap, Networking

核心内容：

  • Web 应用概述
  • HTML 基础结构
  • Bootstrap 框架入门
  • 计算机网络基础（HTTP/HTTPS）

难点：理解浏览器与服务器的交互机制。

Week 2: CSS, Layouts & UI Design

核心内容：

  • CSS 基础语法
  • 盒模型与布局
  • Flexbox 与 Grid 布局
  • 响应式设计
  • 用户界面设计原则
  • 交互体验优化

响应式设计实现方法：

  • 媒体查询：使用 @media 规则根据设备特性应用不同样式
    /* 移动设备 */
@media (max-width: 768px) {
  .container {
    padding: 10px;
  }
}

/* 平板设备 */
@media (min-width: 769px) and (max-width: 1024px) {
  .container {
    padding: 20px;
  }
}

/* 桌面设备 */
@media (min-width: 1025px) {
  .container {
    padding: 30px;
  }
}
  • 流体布局：使用相对单位（%、em、rem、vw/vh）
  • Flexbox 与 Grid：利用弹性布局和网格布局实现响应式设计
  • 响应式图片：使用 max-width: 100% 和 srcset 属性
    <img src="small.jpg" srcset="medium.jpg 768w, large.jpg 1024w" alt="Responsive Image">
  • 字体响应式：使用相对单位和 clamp() 函数
    body {
  font-size: clamp(16px, 2vw, 24px);
}

移动优先设计原则：

  • 从移动设备开始设计
  • 渐进增强：为更大屏幕添加功能
  • 考虑触摸交互：适当的点击区域大小

响应式设计注意事项：

  • 避免使用固定宽度
  • 优化移动设备性能
  • 测试不同设备和屏幕尺寸
  • 考虑网络条件：优化资源加载
  • 确保可访问性：适应不同输入方式

用户界面设计原则：

  • 一致性：保持界面元素的一致性，包括颜色、字体、间距等
  • 层次结构：通过视觉层次引导用户注意力
  • 简洁性：减少视觉噪声，突出核心功能
  • 可访问性：确保所有用户都能使用界面
  • 反馈机制：为用户操作提供及时反馈

交互体验优化：

  • 微交互：添加细微的动画和过渡效果，增强用户体验
  • 响应式反馈：按钮、表单等元素的状态变化
  • 加载状态：提供明确的加载指示器
  • 错误处理：友好的错误提示和恢复机制

设计工具与资源：

  • 设计系统：建立统一的设计规范
  • 色彩工具：Adobe Color、Coolors
  • 排版工具：Google Fonts、Adobe Fonts
  • 原型工具：Figma、Sketch、Adobe XD

难点：CSS 布局的深度理解，特别是 Flexbox 和 Grid 的使用场景，以及如何将设计原则应用到实际项目中。

Week 3: JavaScript, Input and Forms, TypeScript

核心内容：

  • JavaScript 基础语法
  • DOM 操作
  • 表单处理与验证
  • TypeScript 入门（类型系统）

TypeScript 优势：

  • 类型安全：静态类型检查，减少运行时错误
  • 代码提示：提供更好的 IDE 智能提示
  • 可维护性：提高代码可读性和可维护性
  • 渐进式：可以逐步引入，与 JavaScript 兼容

TypeScript 核心概念：

  • 类型注解：为变量、函数参数和返回值添加类型
  • 接口：定义对象的结构
  • 泛型：提高代码复用性
  • 联合类型与交叉类型：增强类型灵活性

代码示例：

// 类型注解
let name: string = "John";
let age: number = 30;

// 接口
interface User {
  id: number;
  name: string;
  email: string;
}

// 函数类型
function greet(user: User): string {
  return `Hello, ${user.name}!`;
}

// 泛型
function identity<T>(arg: T): T {
  return arg;
}

难点：TypeScript 的类型系统与 JavaScript 的动态类型之间的转换。

Week 6: React

核心内容：

  • React 组件化开发
  • JSX 语法与工作原理
  • State 与 Props 的管理
  • 生命周期与 Hooks
  • React 渲染机制

React 工作原理：

  • 虚拟 DOM：React 首先构建虚拟 DOM 树，通过 diff 算法比较前后状态差异，只更新需要变化的部分，提高渲染效率
  • 单向数据流：数据从父组件流向子组件，确保状态管理的可预测性
  • 组件化：将 UI 拆分为独立、可复用的组件，提高代码可维护性

组件化思维：

  • 单一职责原则：每个组件只负责一个特定功能
  • 组合优于继承：通过组件组合构建复杂 UI
  • ** props 传递**：通过 props 在组件间传递数据
  • 状态提升：将共享状态提升到共同的父组件

状态管理：

  • 组件内部状态：使用 useState 管理组件内部状态
  • 复杂状态管理：对于跨组件状态，可使用 Context API 或 Redux
  • 状态更新：React 状态更新是异步的，需要注意批量更新的特性

Hooks 详解：

  • useState：管理组件状态
  • useEffect：处理副作用，如数据获取、订阅等
  • useContext：访问上下文
  • useReducer：处理复杂状态逻辑
  • useCallback：缓存函数，避免不必要的重新渲染
  • useMemo：缓存计算结果，优化性能

代码示例：

// 基本组件结构
function Counter() {
  // 使用 useState 管理状态
  const [count, setCount] = useState(0);
  
  // 使用 useEffect 处理副作用
  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]);
  
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

难点：React 的状态管理与组件通信，理解虚拟 DOM 和 diff 算法的工作原理。

核心内容：

Web 安全深入：

  • HTTP 认证：Basic / Bearer / Digest
  • JWT (JSON Web Tokens)
  • CORS（跨域资源共享）
  • OAuth 2.0 与 OpenID Connect

Week 9: Node.js and Express, MongoDB

Week 10: Web Architectures, Serverless, GraphQL

核心内容：

  • Web 架构模式与演进
  • Serverless 无服务器架构
  • GraphQL 查询语言
  • MongoDB 与 API 实践
  • 现代前端架构趋势

Web 架构模式：

  • 传统 MVC 架构：Model-View-Controller 分离
  • 微服务架构：将应用拆分为独立的服务
  • Serverless 架构：按需执行，无需管理服务器
  • Jamstack 架构：静态站点生成 + API + 边缘部署

Serverless 架构最新发展：

  • FaaS (Function as a Service)：AWS Lambda、Azure Functions、Google Cloud Functions
  • BaaS (Backend as a Service)：Firebase、Supabase
  • 边缘计算：在靠近用户的边缘节点执行代码，降低延迟
  • Serverless 工作流：AWS Step Functions、Azure Logic Apps

GraphQL 优势：

  • 精确数据获取：只获取需要的数据
  • 类型系统：强类型 schema，提供更好的开发体验
  • 单一端点：减少网络请求
  • 实时数据：通过 subscriptions 实现实时更新

现代前端架构趋势：

  • 前端微服务：Module Federation、Micro-Frontends
  • 状态管理新方案：Zustand、Jotai、Valtio
  • Server Components：React Server Components，减少客户端 bundle 大小
  • Edge Runtime：在边缘节点运行前端代码，提高性能

难点：理解不同架构的适用场景，选择合适的架构模式。

Week 14: Frontend Performance Optimization

核心内容：

  • 前端性能优化策略
  • 性能测量与监控
  • 资源优化
  • 代码优化

性能优化策略：

  • 网络优化：

    • HTTP/2 与 HTTP/3
    • 资源压缩（gzip, brotli）
    • 资源缓存策略
    • CDN 加速
    • 预加载（preload, prefetch, preconnect）

  • 资源优化：

    • 图片优化（WebP, AVIF, 响应式图片）
    • 字体优化（字体子集化, 字体加载策略）
    • 代码分割（Code Splitting）
    • 懒加载（Lazy Loading）

  • 代码优化：

    • 减少 JavaScript 体积
    • 优化 CSS 选择器
    • 避免重排（Reflow）和重绘（Repaint）
    • 使用 Web Workers 处理计算密集型任务

  • React 性能优化：

    • 使用 memo, useMemo, useCallback
    • 避免不必要的重新渲染
    • 虚拟列表（Virtual List）
    • 服务端渲染（SSR）和静态站点生成（SSG）

性能测量与监控：

  • 核心 Web 指标：
    • LCP (Largest Contentful Paint)
    • FID (First Input Delay)
    • CLS (Cumulative Layout Shift)
  • 性能工具：
    • Chrome DevTools (Performance, Lighthouse)
    • WebPageTest
    • Google PageSpeed Insights

实践方法：

  1. 性能审计：使用 Lighthouse 进行性能审计
  2. 瓶颈识别：使用 Chrome DevTools 找出性能瓶颈
  3. 优化实施：根据审计结果实施优化
  4. 持续监控：建立性能监控机制

代码示例：

<!-- 图片优化 -->
<img src="image.webp" alt="Optimized Image">

<!-- 预加载关键资源 -->
<link rel="preload" href="critical.css" as="style">
<link rel="preload" href="critical.js" as="script">

<!-- 懒加载图片 -->
<img src="placeholder.jpg" data-src="actual-image.jpg" alt="Lazy Loaded Image">
<script>
  document.addEventListener('DOMContentLoaded', function() {
    const lazyImages = document.querySelectorAll('img[data-src]');
    const imageObserver = new IntersectionObserver((entries, observer) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          const img = entry.target;
          img.src = img.dataset.src;
          imageObserver.unobserve(img);
        }
      });
    });
    lazyImages.forEach(img => imageObserver.observe(img));
  });
</script>

难点：平衡性能优化与开发体验，选择合适的优化策略。
